# Neon Lanes Championship

Neon Lanes is the release-quality evolution of Zanna's 3D Bowling showcase. It
combines regulation ten-pin scoring and physics-driven pin collisions with an
accessible timing release, tactical equipment, persistent progression, and
local arcade modes.

The completed 25-part implementation and corrected game defects are mapped in
[`UPGRADE_AUDIT.md`](UPGRADE_AUDIT.md).

## Modes

- **Quick Game** — a complete single-player ten-frame game.
- **Local Match** — two to four named hot-seat bowlers with individual colors
  and equipment choices.
- **Championship** — a three-rival tour through Riley Chen, Mateo Cruz, and
  Nova King. Every AI delivery uses the same physics, scoring, and camera path
  as the player.
- **Challenges** — Pocket Master, Spare School, and Clutch Tenth events with
  persistent bronze, silver, and gold medals.
- **Practice Lab** — free bowling plus repeatable 7–10, 10-pin, and 3–6–10
  spare drills.
- **Cosmic Party** — short Strike Rush, Low Ball, and accuracy-scored Target
  Bowling sessions with a dedicated neon lighting treatment.

## Controls

| Action | Keyboard | Gamepad |
|---|---|---|
| Move stance | A / D | Left stick |
| Choose target board | Left / Right | D-pad left / right |
| Shape hook | Q/E | Bumpers |
| Toggle spare release | F | X |
| Build power / confirm timing | Space | A |
| Previous / next ball | 1 / 3 | Stick buttons |
| Reset stance, target, and hook | R | — |
| Camera | C | Y |
| Pause / back | P or Escape | Start / B |

Throwing has two independent stages. Hold the throw control to build power,
then release it to lock that value. The accuracy marker starts on its own
cycle; tap throw again near the green center to launch. If the second tap is
lost, the game launches after 1.10 seconds rather than hanging the turn.

Stance and target are deliberately separate. A good shot normally starts from
one board and crosses toward a pocket board; lining the ball up directly with
the head pin is a useful hit but is no longer an automatic strike. The narrow
lane ribbon previews the current delivery and its error width. Timing, power,
hook, equipment control, oil fit, cover family, rack variation, and lane wear
all affect the physical result. F/X flattens axis and hook for corner spares.

Standard assist recovers 38% of human timing error, Full recovers 72%, and Pro
recovers 8%. Assistance never changes pin physics or AI accuracy. Bumpers are
an explicit accessibility setting and default off.

## Difficulty and accessibility

- Casual, Standard, and League alter launch-error width, legal rack variation,
  and AI target bands. Standard is tuned for a learnable pocket strike rate,
  not perfect-shot automation.
- High Contrast affects pins and setup indicators. Large HUD scales the score,
  delivery meter, and toasts while preserving the pocket view.
- Reduced Motion removes camera shake and shortens travel-heavy cinematics and
  replay. Reduced Flash replaces broad flashes with a border pulse.
- House, Sport, and Dry oil use a 13-by-24 live grid. Repeated traffic opens
  friction on that board and carries a bounded amount of oil two cells forward.
- Six balls are sidegrades with distinct speed, hook, control, oil fit, cover
  family, visible rotation stripe, and unlock score.

## Release content

The demo owns its title key art and maple lane texture under `assets/`. The
maple image has separate material and HUD decodes. The existing Metal backend
uses stable procedural boards because of the isolated NL-ZANNA-003 compositor
defect; composition-safe backends may bind the material-only decode. Missing
optional assets resolve through a centralized repository/project/package path
search and fall back procedurally.
Music, ambience, and effects are generated by Zanna's dependency-free audio
synth so the game remains completely self-contained.

Run from the repository root with:

```sh
zanna run examples/games/3dbowling
```

Run all demo release gates against an already-installed/built Zanna binary
without configuring Zanna or invoking CTest:

```sh
ZANNA_BIN=/path/to/zanna examples/games/3dbowling/run_probes.sh
```

On Windows, set `ZANNA_BIN` and run `examples\games\3dbowling\run_probes.ps1`.
The suite checks challenge calibration, identical launch/mask/score replay,
collision-order independence, physical pinfall, oil wear, AI parity, pause and
watchdog recovery, replay scenes, 50 world transitions, corrupt saves, assets,
frame rates, resize layout, accessibility, menus, and visual smoke frames.

The project manifest includes the `assets/` directory in macOS, Linux, Windows,
and portable packages, and uses the original 512-pixel bowling emblem under
`packaging/` as its application icon.

Engine/runtime behavior discovered while building the release upgrade is
tracked with minimal reproductions in
[`known_zanna_issues/README.md`](known_zanna_issues/README.md). Per the current
development constraint, those issues are documented here without modifying or
rebuilding Zanna itself.

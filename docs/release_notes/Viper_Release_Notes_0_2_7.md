### What this release is about

A short hardening cycle continuing v0.2.6. The headline new work is a sustained pass that ends per-frame Graphics3D rendering flicker and closes the open-world ("3D Next Level") streaming gaps, joined by node-animation playback and a substantially upgraded glTF/FBX model importer. Around the 3D work, a repo-wide correctness sweep makes recoverable runtime traps fail closed instead of falling through, promotes IL and codegen invariants from debug asserts to release-mode validation, tightens BASIC/Zia parsing and the Text/Time/IO surface, and modularizes the runtime into focused translation units â€” while the Linux and Windows/MSVC builds both return to green.

- **Graphics3D rendering flicker stabilized.** A sustained pass ends per-frame visible-triangle flicker: queue-order-independent occlusion history with covered-streak gating, camera-depth-fitted shadow cascades, per-camera LOD/impostor hysteresis against a union AABB, and conservative terrain-horizon culling with a two-way far-plane fit. Every backend (software/OpenGL/Metal/D3D11) validates draw-command index ranges and shares one depth-bias scale.
- **glTF/FBX import & node animation (new).** Imported models expose node, object, camera, and morph-weight clips through new `NodeAnimation3D`/`NodeAnimator3D` types, and `Model3D`/`Assets3D` can pull a single clip from an external file (`LoadAnimation`/`LoadNodeAnimation`). The FBX importer gains full model transforms (rotation order, pivots, Z-up), cubic animation curves, PBR texture-slot routing, and async loading, while glTF accessor and skin-weight validation tightens.
- **New Canvas3D authoring helpers.** `DrawMeshWind` sways foliage with height-weighted per-vertex deformation on every backend, `DrawImage2D` blits a `Pixels` image into the 2D overlay (HUD minimaps via `RenderTarget3D.AsPixels`), and `SetFullscreen`/`ToggleFullscreen`/`IsFullscreen` plus `Game3D.Keys.F11` drive native fullscreen. `DrawMeshSkinned` accepts an `AnimController3D` pose, and `Material3D` gains depth-bias and `ShadowMode` (auto/none/cast) controls.
- **Open-world streaming & navmesh tooling (new).** `WorldStream3D` cells load a binary sidecar into their resident-byte budget, and `NavMesh3D.Export`/`Import` round-trip versioned `VNAVMSH2` assets â€” geometry, traversal costs, areas, off-mesh links, obstacles, and agent params â€” while still reading legacy `VNAVMSH1`. Native compressed `TextureAsset3D` uploads drain large mips across frames with KTX2 length validation, and stream manifests resolve `asset://` sidecars through the asset manager.
- **3D runtime fails closed.** Non-finite interpolation, playback, and bone inputs across Skeleton3D, AnimController3D, BlendTree3D, IKSolver3D, and CPU skinning clamp or revert to the bind pose, and scene, transform, raycast, navigation, physics, terrain, water, particle, and Game3D entry points reject bad handles and overflow-prone inputs. Stored camera, node, character, navmesh, and animator references are class-checked before use, so a reused slot is nulled rather than dereferenced.
- **Game3D hardened across threads and capture.** Async model loads read worker-safe request snapshots (never touching runtime strings off-thread) and free staged payloads on shutdown; the world's body/name indices rebuild from the entity registry on detected corruption; tree spawn/despawn are roll-back-exact; and `ScreenshotFinal`/render-target capture finalize a frame with no present side effect, leaked post-FX state, or stale-identity cache hit, with targets reserved against a process-wide budget.
- **Recoverable runtime traps fail closed (repo-wide).** A cppcheck-driven audit makes every recoverable `rt_trap` return a safe sentinel and release its locals, so allocation, overflow, parse, and invalid-handle failures across core, text, crypto, network (HTTP/WebSocket/SMTP/TLS/URL), OOP, process, graphics media/image decoders (AVI/GIF/Theora/PNG/JPEG), and input helpers can no longer fall through into a null dereference, freed storage, or a held mutex. Temp-file, archive, and savedata sidecar names now fail closed when secure OS entropy is unavailable instead of dropping to predictable pid/time values.
- **Allocation, BigInt & protocol edges (closing pass).** BigInt gains checked digit/byte/size arithmetic on every allocation and conversion plus full arbitrary-width two's-complement bitwise and shift semantics, while heap retain/realloc and pool shutdown now validate payload headers under the registry lock and check freed blocks against their owning slab. Protocol paths tighten alongside: HTTP/2 HEADERS framing and zero-frame-size handling, `204`/`304` empty-body suppression, the WSS accept-vs-`Stop` shutdown race, per-policy retry jitter, RSA public-exponent range, SSE content-encoding negotiation, and real unified-diff context windowing.
- **IL, codegen & linker hardening.** The IL builder promotes its debug-only invariants â€” name uniqueness, operand and block-parameter bounds, branch-argument counts, terminated-block appends â€” to release-mode validation, shares one checked-integer-range implementation with CheckOpt, and reports mistyped analysis lookups as actionable errors; CheckOpt now demotes overflow-checked i64 arithmetic to plain ops once range analysis proves it cannot overflow. x86-64/AArch64 instruction selection, encoding, Win64 unwind slots, and AArch64 relocation offsets surface diagnostics instead of asserting â€” on top of the shared frame-layout fix that closes the O1 miscompiles â€” the COFF/Mach-O/PE writers bound their fixed-width name fields, and the native linker hardens relocation, symbol resolution, and PE-image writing.
- **Bytecode VM & support libraries.** The bytecode VM validates memory ranges and indirect callees before dereference, traps instead of asserting on bad branch/switch targets, and bounds runaway programs via `BytecodeVM.setMaxInstructions` and a signal-safe interrupt; the arena and SmallVector guard size and pointer math. An audit of the audio, graphics, and GUI support stack synchronizes sound attach-state under the context lock, recovers the ALSA backend from short writes, replaces the FIFO event-queue overflow with a priority policy that drops transient motion events before close/key-up/focus-lost, and overflow-guards glyph caches, file dialogs, and the code editor.
- **Frontend parsing & language semantics.** BASIC replaces unchecked numeric, line-label, and `SELECT CASE` conversions with overflow-aware token parsing and accepts numbered `DIM` fields inside `CLASS` bodies; Zia restores circular and self binds by treating an in-progress bind as a known dependency rather than a fatal `V1000`; both frontends harden defaults around lowerer state, runtime-registry lookups, import cycles, and completion-cursor bounds.

- **Project loading & packaging.** Project convention detection scans real BASIC/Zia tokens instead of text, rejects sources that escape the project root through symlinks, and gates install hooks and home-Desktop shortcuts behind explicit manifest opt-ins; the PNG, ZIP, tar, PE, and DEB readers and writers tighten chunk ordering, header bounds, and path validation.

# Viper Compiler Platform — Release Notes

> **Development Status:** Pre-Alpha. Viper is under active development and not ready for production use.

## Version 0.2.4 — Pre-Alpha (2026-04-13)

### What this release is about

The largest release in the project's history. Five themes:

- **GPU backend maturity.** Metal and D3D11 reach ~94% parity with the software renderer; OpenGL gains shared infrastructure. 53 per-plan feature implementations across all four backends cover lit textures, spot lights, normal/specular/emissive maps, fog, wireframe, GPU skinning + morph targets, HDR post-FX (bloom / FXAA / tonemap / vignette / color grade), shadow mapping with PCF, instancing, terrain splatting, skybox, and environment reflections.
- **Native toolchain, end-to-end.** `viper build` goes from Zia source to `.exe` / Mach-O / ELF with zero external tools on all three platforms. From-scratch PE/COFF linker, full ELF dynamic-linking, platform-aware Mach-O symbol resolution, Windows ARM64 import-stub emission, and a new `NativeEHLowering` pass that rewrites structured IL EH into ordinary calls before backend lowering.
- **Worker-VM execution model.** Bytecode VMs spawned for `Thread.Start`, `Async.Run`, and HTTP callbacks own a snapshotted `ExecutionEnvironment` (native handlers + runtime-bridge toggles) and can outlive their parent. Per-VM extern registry, resumable trap state, and per-VM interrupt epoch close the lifetime gaps.
- **3D engine + asset pipeline.** Procedural terrain with LOD + frustum culling, Gerstner water, cross-billboard vegetation, glTF 2.0 / STL / OBJ+MTL / FBX loaders, unified `Model3D` asset container, `AnimController3D` state machine, quaternion-based `Physics3D` with CCD + joints, `Collider3D` with 7 reusable shape types, `NavAgent3D` A* pathfinding, 3D audio, video playback (AVI/MJPEG + Theora infrastructure), PBR metallic-roughness materials.
- **2D game engine + asset embedding.** Ten new high-level APIs (Entity, Behavior, Raycast2D, LevelData, SceneManager, Camera.SmoothFollow, AnimStateMachine, MenuList.HandleInput, Config.Load, Tilemap.SetTileAnim) and the compile-time VPA asset-embedding pipeline for zero-file-dependency native executables.

Cross-cutting groundwork: `PlatformCapabilities.hpp` + CMake capability gating (AUTO / REQUIRE / OFF) + unified Unix build scripts + the new `viper install-package` toolchain installer.

### GPU backends

- **Metal — feature-complete** (14 plans, ~47% → ~94% parity). Direct vs GPU-postfx window paths split; offscreen textures + in-flight command buffers cached per render target; packed morph delta / normal-delta buffers cached by `(morph_key, morph_revision, shape_count, vertex_count)` with age-based pruning. Skybox rewritten as a fullscreen triangle reconstructed from `vertex_id` via inverse-projection + inverse-view-rotation. macOS 26 (Tahoe) compatibility via offscreen `MTLStorageModeManaged` readback.
- **D3D11 — 20 features.** 3,173-line HLSL+C rewrite covering diffuse + normal matrix cbuffers, per-frame SRV cache, HLSL cone-attenuation spot lights, wireframe + cull, fog, normal / specular / emissive maps, RTT with resolve, GPU skinning + morph targets, post-FX suite, shared VBO/IBO pool, systematic `FAILED(hr)` validation, shadow mapping (`COMPARISON_LESS_EQUAL` + PCF), per-instance structured buffer, terrain splatting, TextureCube skybox, environment reflections (cubemap + fresnel), morph normal deltas, DOF / motion blur / SSAO / chromatic aberration. New `windows-d3d11` CI job validates in isolation.
- **Shared backend infrastructure.** Per-backend `_shared.{c,h}` pairs factor out constant-buffer layouts, packed morph weight encoding, blend mode / target kind enums, and frame-history tracking. `vgfx3d_backend_utils.{c,h}` adds RGBA unpack, cubemap face decode, mip-count math, normal-matrix derivation, 4×4 inverse.
- **Cross-backend hardening.** `Pixels` and `CubeMap3D` carry a `uint64_t generation` counter bumped by every in-place mutation; GPU caches compare cached vs current generation before reuse (fixes the "modified texture not updating on screen" class of bugs on all backends). Morph-target generation tracking. Canvas3D post-FX state latched at frame boundaries. Resize callback propagates OS events into `Poll()`. GPU screenshot readback with RTT + `backend->readback_rgba()` + software fallback. `InstanceBatch3D` migrated from `realloc` to `calloc + memcpy + free`; swap-remove copies all three parallel arrays. `Mesh3D.transform` uses inverse-transpose of the upper 3×3 for normals on non-uniformly scaled meshes.
- **Fixes.** Metal lit-texture branch didn't sample diffuse; Metal spot lights fell through to ambient; Metal texture + sampler recreated every draw call; Metal clear-color alpha 0.0 with PostFX inactive; software-backend `vgfx3d_show_gpu_layer` duplicate-symbol crash (vtable dispatch); macOS `drawRect:` CGImage blit covering Metal layer content every frame; Water3D wave normals using single-sine `dydz = dydx` (Gerstner model now corrects per-direction).

### Native toolchain

- **PE/COFF linker (new).** `ArchiveReader` extracts `.lib` members; `SymbolResolver` resolves across objects + archives with on-demand member inclusion; `SectionMerger` merges `.text` / `.rdata` / `.data` / `.bss` with alignment padding; `DeadStripPass` removes sections unreachable from entry-point roots; `ICF` folds sections with matching content + relocations; `RelocApplier` handles `IMAGE_REL_AMD64_*`; `PeExeWriter` produces PE32+ executables with `.idata` import tables and optional headers.
- **ELF dynamic linking (new).** `ElfExeWriter` rewritten for full dynamic-link: `PT_INTERP`, `PT_DYNAMIC`, `.dynsym`, `.dynstr`, `.hash` (SYSV), `.rela.dyn` with `R_X86_64_GLOB_DAT`. Linux x86_64 programs link natively against libc / libm / libpthread / libX11 / libasound without falling back to the system linker. `planLinuxImports()` classifies dynamic symbols into shared libraries via prefix/exact-match tables; `generateDynStubsX8664()` emits 6-byte `jmpq *__got_sym(%rip)` stubs.
- **Linker hardening.** `BranchTrampoline` uses boundary-based AArch64 placement with ±128 MB displacement validation. `SectionMerger::assignSectionVirtualAddresses()` is public (no duplicate VA-assignment). `SymbolResolver::resolveSymbols()` takes a `LinkPlatform` parameter with per-platform prefix tables. `writeCheckedRel32()` validates signed-32 range. Multi-section COFF writer. Mach-O ObjC section flags (`S_CSTRING_LITERALS` / `S_LITERAL_POINTERS` / `S_ATTR_NO_DEAD_STRIP` / `S_COALESCED`) with log2-power-of-two alignment. Mach-O symbol-name fallback (plain + underscore-stripped + underscore-prefixed) and `OBJC_CLASS_$_` / `OBJC_METACLASS_$_` / `OBJC_EHTYPE_$_` normalization. Mach-O symbol-index sentinel fix (slot 0 no longer means "not found"). Windows ARM64 import stubs (`ADRP` / `LDR` / `BR` with `kPageRel21` / `kPageOff12L`). Table-driven import plans. `DynamicSymbolPolicy.hpp` extracted from `SymbolResolver`. Proper zerofill (`S_ZEROFILL` / `SHT_NOBITS`). Dead-strip applies to every object. String-dedup compaction. ICF cross-object resolution through the global symbol table. `-lshell32` added on Windows for `DragQueryFile`/`DragAcceptFiles`.
- **NativeEHLowering (new pass, 683 LOC).** Rewrites structured IL EH markers (`EhPush` / `EhPop` / `ResumeSame` / `ResumeNext` / `TrapErr`) into ordinary calls and branches before backend lowering. Both x86_64 and AArch64 `CodegenPipeline` invoke it before MIR lowering. 171-line unit test validates all transformations.
- **Fixes.** ELF writer emitted `STT_FUNC` for rodata; COFF `validateCoffRelocationAddend()` diagnostic for unsupported addends and `.pdata` function length field emitted (was 0); missing `libviper_rt_game.a` in archive discovery.

### VM & runtime

- **Worker VM execution environment.** `ExecutionEnvironment` (runtime-bridge toggle + threaded-dispatch toggle + native-handler map) captured / applied / copied across VMs. `Thread.Start`, `Async.Run`, and the HTTP-server request dispatcher all call `copyExecutionEnvironmentFrom(parent)` on workers so each owns its own native-handler copy.
- **Per-VM extern registry.** Resolution order: per-VM → process-global → built-in `RuntimeDescriptor`. Per-registry mutex for both registration and lookup. `PerVMExternRegistryTests` covers per-VM-only registrations, override of process-global names, and sibling-VM isolation.
- **Resumable trap state.** `BytecodeVM::TrapRecord` snapshots kind, error code, fault PC, next PC, fault line, value / call / EH stacks, and live `alloca` storage — a trap escaping `try/catch` in one frame re-raises cleanly in the caller.
- **Per-VM interrupt epoch.** `std::atomic<uint64_t>` epoch replaces shared `atomic<bool>`; each VM tracks its own observed cursor. Multiple coexisting VMs (REPL + background thread, embedding host + child eval) observe Ctrl-C independently. `clearInterrupt` advances a cleared-epoch watermark.
- **Object lifetime.** Zeroing weak references (`rt_weak_load`/`rt_weak_store` through `rt_weakref_*` handles with `rt_weakref_is_handle` discriminator for raw pointers). `Object.ToString` returns a fresh `rt_string` for string targets and validates the heap header before dereferencing the vptr. `MessageBus.Publish` uses snapshot semantics so subscribers can unsubscribe themselves mid-publish.
- **CALL_NATIVE widened.** `[opcode:8][nativeIdx:8][argCount:8]` (max 255) → `[opcode:8][argCount:8][nativeIdx:16]` (max 65,535). Bytecode format version bumped 1 → 2.
- **Trap + error plumbing.** `rt_trap_set_ip` / `rt_trap_get_ip` store the native IP at trap time (`TrapGetIp` in `runtime.def`). `rt_err_to_trap_kind` maps `RtError` codes (overflow=1, invalid cast=2, domain=3, bounds=4, file-not-found=5, …). `rt_trap_error_make` / `rt_trap_raise_error` combine message storage + field classification + trap invocation.
- **Input subsystem.** `Keyboard.CapsLock` queries real OS state every call (`GetKeyState(VK_CAPITAL)` / `CGEventSourceFlagsState` / `XkbGetIndicatorState`) instead of toggle-counting. `Mouse.SetPos(x, y)` dispatches to `CGWarpMouseCursorPosition` / `SetCursorPos` / `XWarpPointer`. Double-precision wheel deltas (`Mouse.WheelXF` / `WheelYF`). Canvas destruction detaches input bindings via `rt_keyboard_clear_canvas_if_matches` / `rt_mouse_clear_canvas_if_matches`.
- **Fixes.** `rt_exc_is_exception()` accepted any non-null pointer; OOP destructor chaining missing (derived dtors never called base); OOP refcount imbalance (`NEW` objects ended at refcount 2); TLS RSA-PSS SHA-384/512 hashed with SHA-256; bytecode VM `LOAD_I32` / `LOAD_STR_MEM` / `STORE_STR_MEM` routed to unimplemented-trap; POSIX `ProcessIsolation::runIsolated()` blocked forever (no timeout, now WNOHANG poll + `SIGKILL`); `Pixels.Blur` / `Pixels.Resize` unpacked ARGB instead of RGBA (R and A swapped on non-opaque alpha); Windows `Viper.Threads` test disabled but runtime implemented.

### Compiler & codegen

- **Zia language features (7 new).** Type aliases (`type Name = TargetType;`); variadic parameters (`func sum(nums: ...Integer)`, collected as `List[Integer]`); shift operators (`<<` / `>>`, arithmetic right shift, correct precedence between additive and comparison); compound bitwise assignments (`<<=` / `>>=` / `&=` / `|=` / `^=` desugared via read-op-store); single-expression functions (`func f(x: T) -> R = expr;`); lambda expressions (`(x: T) => expr` and `() => expr`); polymorphic `is` matching any subclass via `collectDescendants()`. Keyword rename: `entity` → `class`, `value` → `struct` across all source, tests, REPL, LSP, runtime GUI, docs, VS Code extension, and website.
- **Zia frontend.** `rtgen` emits full ABI-shaped extern signatures (so string-returning runtime methods compare via `Viper.String.Equals` instead of pointer-`ICmpEq`); typed return metadata (`obj<ClassName>`, `seq<str>`) for 100+ factory/conversion/collection methods; new `concreteRuntimeReturnClassQName` API + BASIC `inferObjectClassQName` recursive type tracer; runtime property setters (`ctrl.VY = value` calls the setter); `final` enforcement in sema (V3000 error on reassignment of `final` vars, for-in loop vars, match pattern bindings) + lowerer safety net; `catch(e)` binding via `rt_throw_msg_set`/`_get` TLS + `ErrGetMsg` IL opcode; `String.Contains()` alias to `StrHas`; `main()` entry-point parity with `start()` (interface itable init + global initializers); optional struct boxing via `emitBoxValue` (no more dangling stack pointers); completion runtime APIs (`Viper.Zia.Completion.Check` / `.Hover` / `.Symbols`) via error-tolerant partial compilation.
- **BASIC frontend.** `FoldBuiltins.cpp` evaluates 25+ builtins (`ABS`, `INT`, `SGN`, `SQR`, `LOG`, `EXP`, `SIN`, `COS`, `TAN`, `ATN`, `ASC`, `CHR$`, `LEN`, `LEFT$`, `RIGHT$`, `MID$`, `STR$`, `VAL`, `STRING$`, `SPACE$`, `LCASE$`, `UCASE$`, `LTRIM$` / `RTRIM$` / `TRIM$`) at compile time when arguments are constant.
- **x86_64 backend.** CoffWriter cross-section symbols (rodata first; text relocations redirect to defined entries, fixes LNK2001). `mapRuntimeSymbol()` translates IL runtime names to C names. Operand materialisation for `TESTrr` and `call.indirect` immediates (no more `bad_variant_access`). SETcc REX prefix correct for SPL/BPL/SIL/DIL. SSE RIP-relative `MOVSD` encoding. Spill-slot reuse disabled where cross-block liveness wasn't accounted for. Branch relaxation (short JMP + short Jcc, 2 bytes each). Pipeline decomposition (`legalizeModuleToMIR` / `allocateModuleMIR` / `optimizeModuleMIR` / `emitMIRToAssembly` / `emitMIRToBinary` as public APIs). `CallArgLayout` + `FrameLayoutUtils` extracted as shared utilities. System linker shell-out removed (~330 lines of `cc` / `ld` invocation); native linker is default, `--system-link` deprecated.
- **AArch64 backend.** Register-allocator protected-use eviction (`protectedUseGPR_` / `protectedUseFPR_`) prevents clobbering current-instruction sources — fixes `BowlingGame.init` GEP-base crash under register pressure. `isUseDefImmLike` operand roles classify operand 0 as DEF-only (AddRI, SubRI, …). FPR load/store classification in `isMemLd` / `isMemSt`. Clean FPR spill-slot reuse for rodata values surviving calls in caller-saved FPRs. Live-out spill fix (`isLiveOut()` query prevents premature release; forward end-of-block scan). `TrapErr` materialises the message string into x0 — native `throw "msg"` no longer produces empty diagnostics. Error-field extraction via TLS (`rt_trap_get_kind/code/line`) for typed `catch(e: DivideByZero)` in native code. Apple M-series scheduler tuning (FP divide 3→10, int divide 3→7, FP multiply 3→4 — matches Firestorm latencies). Join-phi coalescing peephole (Pass 4.88) eliminates stack round-trips at CFG join blocks. CBR edge-block generalization (edge blocks emit whenever branch arguments are present, not only when both arms target the same block). Loop phi-spill multi-block latch fix. Pipeline decomposition via `PassManager` (Scheduler + BlockLayout passes at O1+). Virtual register space partitioned (`kFirstVirtualRegId=1`, `kPhiVRegStart=40000`, `kCrossBlockSpillKeyStart=50000`). Branch relaxation (`computeFunctionLabelOffsets` + `measureInstructionSize` + `estimateFunctionSize`; new `A64CondBr19` relocation for conditional branches exceeding ±1 MB). `FrameBuilder` spill-epoch guards prevent stale-offset reuse.
- **IL optimizer.** EarlyCSE / GVN textual-ordering fix: `AvailableExpr{value, block}` + `isTextuallyAvailable()` prevents replacements that would create textual use-before-def. Mem2Reg single-predecessor + entry-block fix (forward from predecessor or use typed zero constant; no unbindable synthetic block parameters). SCCP preserves dominators + loop info + CFG instead of conservatively invalidating.

### Asset embedding (VPA)

Compile-time asset packaging for zero-file-dependency native executables. Assets compile into a binary blob at build time and embed into the executable's `.rodata`.

- **Project directives** in `viper.project`: `embed <path>` (raw bytes), `pack <path>` (VPA container framing), `pack-compressed <path>` (DEFLATE).
- **Toolchain.** `AssetCompiler` (233 LOC) reads directives, resolves paths, invokes `VpaWriter` (230 LOC, magic header + file table + concatenated data + optional DEFLATE). `VpaReader` (`rt_vpa_reader.c`, 375 LOC) mounts VPA archives at runtime with multi-archive support. Both x86_64 and AArch64 `CodegenPipeline` inject the VPA blob as `viper_asset_blob` / `viper_asset_blob_size` globals in `.rodata`. `Path.ExeDir()` resolves the executable directory cross-platform (`_NSGetExecutablePath` / `/proc/self/exe` / `GetModuleFileName`).
- **Runtime API (`Viper.IO.Assets`).** `Load(name)` / `LoadBytes(name)` / `Exists(name)` / `Size(name)` / `List()` / `Mount(path)` / `Unmount(path)`. Resolution order: embedded blob → mounted VPA archives → filesystem fallback relative to the executable directory. 9 VPA format tests + 9 asset manager tests.
- **Fixes.** VAPS DEFLATE double-free crash. `.lnk` shortcuts missing LinkInfo structure.

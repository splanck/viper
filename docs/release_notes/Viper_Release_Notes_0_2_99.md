# Viper Compiler Platform — Release Notes

> **Development Status:** Pre-Alpha. Viper is under active development and not ready for production use.

## Version 0.2.99 — Pre-Alpha (DRAFT — unreleased)

### What this release is about

A cleanup-and-packaging follow-up to the v0.2.7 hardening cycle, pairing a settled public API and a codegen optimization round with a major 3D rendering push, a broad ViperIDE workbench pass, and a real installer story.

The runtime's public surface finishes settling into its final shape — one canonical name for everything, terse abbreviations spelled out, and, most visible to your code, every recoverable failure handed back as a value you can inspect. Reading past the end of input, a decrypt that doesn't authenticate, a lookup that finds nothing, a parse that fails: each returns an `Option` or a `Result` instead of a null, a `-1`, or a silent side channel. The older shapes stay as compatibility aliases, and `viper --dump-runtime-api` now emits a full machine-readable contract — types, ownership, stability, and security notes — so editors and tools can consume the surface without guessing.

The 3D renderer takes a large step toward modern real-time lighting: image-based lighting brings spherical-harmonic ambient and prefiltered specular cubemaps, a clustered forward+ path bins point and spot lights into camera-space froxels so a scene can carry dozens at once, and screen-space reflections, soft particles, and temporal anti-aliasing land on top — every one carried across the Metal, OpenGL, D3D11, and software backends and fed by a from-scratch Zstandard decoder for compressed KTX2 textures. A dedicated first-person pass then layers on view-model rendering, point-light shadows, height fog, and an auto-exposure/color-LUT/sun-shaft post-FX stack, with raw-mouse input and Doppler-fed spatial audio to match — and the D3D11 backend absorbs a robustness pass so none of that new state can crash a draw with invalid numerics. A read-only diagnostics layer then makes the renderer's fallbacks, dropped draws, and mesh-memory budgets inspectable from your program, with non-trapping probe forms for the features a backend may or may not support. A closing depth-precision pass flips GPU scene rendering to reversed-Z, stabilizes cascaded shadows, and rasterizes true triangle silhouettes for occlusion culling, while the asset pipeline learns to decode compressed glTF geometry and textures and build automatic LOD chains. A final production pass then completes that zero-dependency path — full sequential and edgebreaker Draco decode, an extended PBR material set, BC6H HDR textures, and skeletal rigs scaled to 1,024 bones — all conditioned offline through a new `viper asset` tool. A closing gameplay layer turns the renderer into a character-action toolkit: a third-person camera and controller with target-lock, Character3D dynamics, combat and ragdoll, cloth, cinematics, AI behavior trees, interaction, minimaps, world persistence, and quests land alongside a fresh baked-GI, reflection-probe, and procedural-sky lighting round — twenty-seven runtime subsystems, each with a VM==native probe (ADRs 0074–0100).

Underneath, codegen gets faster: dense switches compile to jump tables, small-integer overflow checks lower to native flag-setting instructions, and a new range analysis proves when a checked add or divide can safely become a plain one. Packaging grows up — the installer now ships ViperIDE, standalone applications package as AppImage, RPM, DMG, and Windows installers, and a release pipeline (ADR 0073) structurally verifies, checksums, and manifests every artifact before it ships while the repository consolidates under `src/` — and the native linker resolves cleanly on both Linux and Windows/MSVC toolchains, now rejecting imports exclusive to another platform. A closing hardening pass clamps every D3D11 shader-facing value before upload, lets ViperIDE's terminal, debugger, build, and source-control panels degrade cleanly under a flood of child output, and makes `Viper.Crypto` validate every input up front — including a ChaCha20-Poly1305 fix that restores interoperable AEAD tags.

- **The public API settles on one canonical name.** Aliases and duplicates are gone and the generator refuses to mint new ones — collection sizes read as `Count`, semantic lengths as `Length`, boolean probes return true/false, and `Entity3D` positions are plain properties, with both languages, ViperIDE, the demos, and the docs rebound to the smaller surface.
- **Recoverable failures come back as values.** Terminal reads, diagnostics, collection and channel pops, decryption, HTTP/REST/SMTP sends, data-format parsing, searches, shell commands, and game and 3D queries all gain `Option`- and `Result`-returning forms, so absence and error are values instead of nulls, sentinels, or side channels; the older forms remain for compatibility.
- **Terse names spell themselves out, and the sharp edges are labelled.** `LeadZ` becomes `CountLeadingZeros`, `NumSci` becomes `Scientific`, and factories drop redundant `New` prefixes, while manual memory, trap-state mutation, legacy ciphers, and the TLS-verification bypass move into plainly named `Runtime.Unsafe`, `Crypto.Legacy`, and testing-only homes so nothing dangerous hides behind an innocent name.
- **A machine-readable API contract (ADR 0027).** `viper --dump-runtime-api` keeps its shape and adds parsed types, ownership, stability, capability, and security metadata for every entry, while a new internal-only marker keeps implementation helpers out of the public dump.
- **Crypto input validation, and a ChaCha20-Poly1305 interop fix.** The AEAD path clamps its Poly1305 key with the RFC 8439 26-bit limb masks — it previously produced non-interoperable tags for some one-time keys — and every `Viper.Crypto` entry point validates its handles before use and halts after a validation trap instead of continuing with sentinel buffers.
- **Modern 3D lighting arrives (new).** Image-based lighting adds spherical-harmonic ambient and prefiltered specular cubemaps; a clustered forward+ path bins lights into camera-space froxels (`Canvas3D.ClusteredLighting`, a 64-light GPU budget, with a flat 16-light software path and a `VIPER_3D_CLUSTERS=0` escape hatch); and screen-space reflections (`Material3D.SsrEnabled`), soft particles, and temporal anti-aliasing layer on top — all across the Metal, OpenGL, D3D11, and software backends, fed by a from-scratch Zstandard decoder for compressed KTX2 textures.
- **A first-person game-engine pass (new).** The 3D runtime gains view-model rendering, point-light shadows with slot budgeting and telemetry, height fog, and an auto-exposure/color-LUT/sun-shaft post-FX stack, plus nine-slice HUD draws and anti-aliased 2D text; input adds raw/relative mouse and gamepad-merged look; spatial audio gains pitch, low-pass occlusion, sidechain ducking, and Doppler; and authoring adds deterministic mesh simplification and LOD, Quickhull convex-hull colliders, CCD, and tiled navmesh bake/rebuild.
- **Game3D physics, water, and test knobs.** A standalone terrain can join physics collision, water planes recenter over off-origin terrain, GPU post-FX composites the HUD into captured frames, `Game.UI` widgets draw against either a 2D `Canvas` or a `Canvas3D` (ADR 0065), and new environment switches let display-requiring tests render without stealing window focus.
- **The D3D11 backend gets a robustness pass.** Every shader-facing constant buffer — camera, skybox, scene, post-FX, TAA, SSR, IBL, and clustered-light state — is clamped to finite, in-range values before upload; compressed-texture support expands to first-class BC1/BC3/BC4/BC5 alongside BC7, ASTC, and ETC2 with per-format block validation; and IBL cubemaps, bloom mip chains, and depth bindings are validated so invalid state can no longer leak into a frame. The backend's oversized HLSL sources move to chunked storage joined lazily on first use, and the per-scene constant buffer is explicitly padded with compile-time `float4`-alignment assertions, so MSVC builds the backend cleanly.
- **Canvas3D observability and non-trapping probes (new; ADRs 0069–0071).** Read-only accessors surface what a frame used to hide — `Canvas3D.IsAvailable`, a `BackendFallbackReason` string, per-frame event- and instanced-draw drop counts, and mesh snapshot/retained-byte budgets (`Mesh3D.RetainedBytes`, kept distinct from drawable residency) — while `Canvas3D.TrySetClusteredLighting` and now-runtime-accurate `BackendSupports("hdr-scene"/"taa")` let code query a capability instead of trapping on a backend that lacks it. Alongside, a robustness round makes the physics broadphase deterministic across VM and native, prepares image-based lighting lazily on the first PBR draw, sanitizes light and shadow-bias inputs, and scales the software rasterizer to sixteen workers.
- **A 3D flicker-and-correctness pass (new).** A Metal frames-in-flight throttle and per-draw transient-buffer ring fix two structural races — instanced batches rendering with stale transforms and dynamic geometry rewritten while the GPU still read it — and `Quat.FromEuler`/`Entity3D.SetRotationEuler` now rotate about the documented pitch→X/yaw→Y/roll→Z axes. Grass thins monotonically through a spatial grid, world streaming gains eviction hysteresis and a reload cooldown so boundary cells stop thrashing, depth converges across backends (OpenGL joins the strict-less compare, GPU depth-bias is recalibrated to the software reference, decal bias scales with size), and instanced forests, transparent draws, and end-of-life particles stop popping as a block. Hot paths get cheaper — TRS-space skeleton blending, once-per-draw radix keys, batched `SceneNode.SetTransform`, and allocation-free component getters cut per-object VM crossings and GC churn — and `Ray3D.IntersectTriangleCull` adds an optional backface-culling picking test.
- **Reversed-Z depth and triangle-accurate occluders (new).** GPU scene passes render with reversed-Z float depth on every backend that advertises it (Metal, D3D11, OpenGL) — the projection's z row is negated at the canvas seam, depth clears to `0` and compares with the `Greater` family, and the skybox sits at the far plane — so float precision concentrates in the distance and open-world clip ranges stop shimmering. The software backend stays standard as the deterministic golden reference, and `Camera3D`, `Mat4.Perspective`, and every public math API keep the standard convention. The Hi-Z occluder grid now rasterizes each eligible opaque draw's real triangles into a 256×256 per-texel depth buffer (perspective-correct, capped at 1,024 triangles per draw and 8,192 per frame), so near walls and rotated geometry finally cull what hides behind them instead of leaning on a conservative AABB rectangle.
- **Shadow quality, frame pacing, and render interpolation (new).** Directional shadows default to two cascades on capable backends with bounded camera coverage, logarithmically weighted splits, cross-cascade blending, distance fades, and hardware comparison sampling, while shadow-aware traversal retains off-screen casters and the skybox moves behind the opaque pass for early depth rejection. New `Canvas3D` surfaces expose shadow distance, vsync present pacing across Metal/D3D11/OpenGL, and a render scale that lets the software renderer rasterize at reduced resolution before upscaling; `World3D.RenderInterpolation` blends pre-step entity poses for smooth fixed-step rendering without disturbing simulation state; and portable scene-depth probes read back one frame late (async on GPU, synchronous in software) to drive stall-free, temporally smoothed lens-flare occlusion.
- **Compressed glTF import, automatic LODs, and material variants (new).** The from-scratch importer decodes `EXT_meshopt_compression` streams and their octahedral/quaternion/exponential filters, `KHR_mesh_quantization` attributes and morph deltas, full sequential and edgebreaker `KHR_draco_mesh_compression` geometry, and `KHR_texture_basisu` textures over in-tree BasisLZ/ETC1S and UASTC KTX2 decoders — rejecting malformed streams and unsupported point clouds with named recoverable diagnostics. A shared QEM `GenerateLODs` decimates each unique mesh once and installs bounded LOD chains that survive instantiation, `SceneAsset` applies and reverses `KHR_materials_variants`, and `AssetDiagnostics3D.GetImportReport` returns structured JSON for skipped primitives, truncated skin influences, ignored extensions, and baked spline channels.
- **Deterministic vegetation and a second 3D throughput round (new; ADR 0072).** `Vegetation3D.SetSeed` pins scatter to a reproducible layout, `Canvas3D`/`RenderTarget3D` grow to 16,384 px with HDR emissive color preserved through upload and partial per-mip BC7 fallback, and `NavMesh3D` bake and pathfinding scale up with struct-derived voxel budgets, cached off-mesh adjacency, and full funnel corridors. Underneath, shared retained-reference slot helpers, a portable MSVC/GCC/Clang atomics surface, reusable light and scratch arenas, and precomputed shadow-caster indices cut per-frame allocation and standardize cross-backend cleanup.
- **The 3D asset pipeline reaches production (new).** Signed and unsigned BC6H HDR blocks and the extended PBR set — clearcoat, IOR, sheen, anisotropy, transmission, and volume across all four backends — close out the zero-dependency glTF/KTX2 path, while animation import scales to production rigs: skeletons up to 1,024 bones partitioned into 256-slot draw palettes, optional influence-5–8 CPU side streams, authored CUBICSPLINE tangents, tolerance-based clip compression, and version-three VSCN baking. New runtime surfaces — `SceneAsset.LoadWithOptionsEx`, `Mesh3D.CompactStreams`, `Canvas3D.DrawInstancedSkinned`, texture-residency streaming with telemetry, 48-byte compact vertex streams, Radiance HDR panorama cubemaps, and one shared split-sum environment-BRDF lookup — sit behind new offline `viper asset bake` and `viper asset validate` conditioning.
- **A third-person action-game runtime suite (new; ADRs 0074–0100).** Twenty-seven subsystems extend the 3D stack from a rendering/physics core into a character-action toolkit: a spring-arm third-person controller with target-lock, Character3D dynamics over moving platforms, combat volumes with health and ragdoll blend-out, slow-motion time control, and rail-camera/timeline cinematics with dialogue and facial/voice metering; Verlet cloth, physics materials driving footstep surface events, AI perception with behavior trees, interaction prompts, minimap markers, world persistence, and a quest system; and a world-and-lighting round — async streaming with HLOD proxies and GPU-skinning telemetry, terrain holes and splat auto-blend, height fog and sun inscattering, baked-GI lightmaps, reflection probes, and procedural sky with time-of-day. Each subsystem lands wired through the IL surface registries with a matching ADR and a VM==native probe.
- **Faster switches and overflow checks.** Dense `switch` statements compile to a single bounds check plus a jump table on both backends, and 32- and 16-bit overflow-checked arithmetic now lowers to native flag-setting instructions instead of widening to 64 bits.
- **Checked arithmetic is demoted when it's provably safe (ADR 0026).** A whole-function range analysis proves when an overflow-checked add, subtract, multiply, or divide can never trap and rewrites it to the plain, faster form — sharing one range implementation with the verifier — and recovers power-of-two modulo bounds after peephole lowering, so `expr % 2^k` keeps verifying at `-O2`.
- **The IL spec version is now 0.3.0 (ADR 0064).** The `il` banner catches up to the surface these codegen rounds added — `switch.i32` jump tables, branchless `select`, and checked/narrow arithmetic — and snapshot builds are now date-stamped (`0.2.99.20260704`).
- **Zia gains target-typed list combinators (new).** `map`, `filter`, `reduce`, `firstWhere`, `any`, `all`, and `sum` target their lambda arguments from the receiver's element type and lower to inline loops over the list runtime. A shared precedence-climbing parser and a language-correctness pass land alongside — String `for`-in, range match patterns, `final`/`let` fields, chained-comparison rejection, and MCP `structuredContent` for diagnostics, completions, symbols, and runtime search.
- **Packaging becomes a real release story (ADR 0025).** The toolchain installer now builds and ships ViperIDE, and standalone applications package as first-class AppImage, RPM, DMG, and Windows installers with desktop launchers, branded icons, and post-build verification, plus a `viper package --dry-run --json` for scripting.
- **A release pipeline and consolidated repo layout (ADR 0073).** Package builders route through a shared `PkgVerify` that structurally validates PE, Mach-O, and ELF artifacts; `viper install-package` rejects a target or architecture that conflicts with the staged binary's object header and requires a macOS universal stage to carry valid, in-bounds slices instead of a relabeled single slice; the Linux self-extracting toolchain format is renamed AppImage → `linux-bundle` (`.run`) to match what it is; and every artifact emits a SHA-256 checksum and a machine-readable manifest gated on structural and lifecycle verification. The repository consolidates under `src/` — the top-level `tests/` fixture tree and `viperide/` tree move to `src/tests/fixtures/` and `src/viperide/`, with CMake registration, test helpers, and e2e scripts repointed at the new locations.
- **Native builds are stable on Linux and Windows, with a tightened linker policy.** Static archives stop pulling libgcc/libstdc++ helpers the native linker can't resolve, the MSVC C++ runtime archives resolve from `VCToolsInstallDir`, and the CRT-less PE path initializes from `main` and exits through `ExitProcess` (mirrored on AArch64). The linker now rejects dynamic imports exclusive to another platform, completes Windows AArch64 PE output, signs Mach-O ad-hoc with a dependency-free SHA-256 (x86-64 Rosetta included), preserves ELF `p_filesz` zero-fill, and broadens its known libc and Windows runtime imports — now including the foreground-activation, raw-input, focus, message-pump, and `log2f` mappings the new GUI and input paths need. The generated AArch64 and x86-64 codegen tables are checked into the tree, so a clean source build consumes them directly instead of regenerating.
- **ViperIDE source control, navigation, and document state (new).** An asynchronous `Process`-backed git layer keeps porcelain-v2 status, staged actions, and push/pull off the UI thread with concise failure hints; BASIC editing gains go-to-definition, references, rename, call hierarchy, and workspace symbols over the runtime `FileIndex`; and open documents track external change, delete, and rename with state-specific save-and-reload recovery.
- **ViperIDE terminal, debugging, and fail-safe child IO (new; ADR 0066).** The integrated terminal becomes a real terminal — CSI addressing, alternate-screen buffers, 24-bit and 256-color SGR, xterm key sequences, and a bounded replay buffer for sessions started while hidden — while new non-trapping process and PTY read-result APIs surface runaway child output with a truncation marker instead of trapping the workbench. The debugger adds command-palette watch management and a grouped Variables view with a VM-backed expander for List, Seq, and Map values whose expansion survives across stops.
- **ViperIDE command input and named toolbar icons (new; ADR 0067).** A reusable in-editor command overlay replaces external prompts for Go To Line, Add Watch, Rename Symbol, Extract Local/Function, and workspace-symbol lookup, while toolbar and activity-bar buttons trade glyph labels for semantic named icons drawn through the vector renderer.
- **ViperIDE becomes a multi-document, multi-root workbench (new).** Per-document buffers preserve undo, cursor, and scroll across tab switches; split panes, crash-recovery swap files, a welcome and recents surface, and inline SCM diff gutters fill out the editing surface; and workspaces advertise multiple roots so search, indexing, and watchers span secondary folders. Underneath, the monolithic `AppShell` splits into focused modules, and a performance pass adds damage-region partial repaint plus an incremental `Viper.Zia.Document` mirror that syncs edit deltas instead of re-sending the whole buffer.
- **GUI apps can claim OS focus (new; ADR 0068).** `Viper.GUI.App.Activate` — with `App.Focus` routed through it — requests native foreground and menu ownership across macOS, Windows, Linux, and the mock backend; ViperIDE uses it at startup and adds an adaptive `PollWait` cadence that stays live while builds, debugging, terminals, overlays, or indexing are active.

Demos and docs tracked the work. ASHFALL — a new nine-level first-person campaign written entirely in Zia over the engine surface above, with hybrid CC0-and-procedural art — joined the set as the flagship for the first-person pass and grew a full combat and campaign layer: staged loadouts, grenades, melee and flashlight, authored enemy flanking, objective beacons with nearest-hostile CLEAR/KILL guidance, and nine redressed levels under a deterministic probe lane. Ridgebound became a small survival game (campfires, forage, warmth-driven stamina, weather-aware image-based lighting) atop its character-controller and boulder physics, 3D Bowling grew into Neon Lanes Championship 2.0 — Quick Game, two-to-four-player local matches, a physics-driven championship tour, medal challenges, and a broadcast-presentation pass — and a fresh `game3d` showcase drives the clustered-lighting and reflection paths. Zia chess became the canonical release-style demo — settings, multi-slot save, clocks, puzzles, FEN/PGN clipboard export, procedural audio, plus high-contrast and large-text modes — and retired its BASIC predecessor, while Xenoscape shipped its 1.0 — a ten-region action Metroidvania with an eighty-room world graph, a permanent upgrade economy, three-slot save migration, Time Trials, Boss Rush, New Game+, and true endings. ViperSQL and baseball moved onto the new terminal APIs, fresh IL benchmark kernels exercise the codegen round, and the 3D, packaging, CLI, and library docs picked up the new lighting surface, installer payloads, canonical names, the API contract, the non-trapping process/PTY read-result APIs (ADR 0066), the Canvas3D diagnostics, clustered-lighting probe, and mesh retained-bytes surfaces (ADRs 0069–0071), the reversed-Z depth model, triangle-accurate occluders, compressed-glTF import, and deterministic vegetation seeding (ADR 0072), the third-person action-game runtime suite and its quest system (ADRs 0074–0100), and the release pipeline and consolidated `src/` layout (ADR 0073).

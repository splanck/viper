### What this release is about

A cleanup-and-packaging follow-up to the v0.2.7 hardening cycle, pairing a settled public API and a codegen optimization round with a major 3D rendering push, a broad ViperIDE workbench pass, and a real installer story.

The runtime's public surface finishes settling into its final shape â€” one canonical name for everything, terse abbreviations spelled out, and, most visible to your code, every recoverable failure handed back as a value you can inspect. Reading past the end of input, a decrypt that doesn't authenticate, a lookup that finds nothing, a parse that fails: each returns an `Option` or a `Result` instead of a null, a `-1`, or a silent side channel. The older shapes stay as compatibility aliases, and `viper --dump-runtime-api` now emits a full machine-readable contract â€” types, ownership, stability, and security notes â€” so editors and tools can consume the surface without guessing.

The 3D renderer takes a large step toward modern real-time lighting: image-based lighting brings spherical-harmonic ambient and prefiltered specular cubemaps, a clustered forward+ path bins point and spot lights into camera-space froxels so a scene can carry dozens at once, and screen-space reflections, soft particles, and temporal anti-aliasing land on top â€” every one carried across the Metal, OpenGL, D3D11, and software backends and fed by a from-scratch Zstandard decoder for compressed KTX2 textures. A dedicated first-person pass then layers on view-model rendering, point-light shadows, height fog, and an auto-exposure/color-LUT/sun-shaft post-FX stack, with raw-mouse input and Doppler-fed spatial audio to match â€” and the D3D11 backend absorbs a robustness pass so none of that new state can crash a draw with invalid numerics. A read-only diagnostics layer then makes the renderer's fallbacks, dropped draws, and mesh-memory budgets inspectable from your program, with non-trapping probe forms for the features a backend may or may not support. A closing depth-precision pass flips GPU scene rendering to reversed-Z, stabilizes cascaded shadows, and rasterizes true triangle silhouettes for occlusion culling, while the asset pipeline learns to decode compressed glTF geometry and textures and build automatic LOD chains.

Underneath, codegen gets faster: dense switches compile to jump tables, small-integer overflow checks lower to native flag-setting instructions, and a new range analysis proves when a checked add or divide can safely become a plain one. Packaging grows up â€” the installer now ships ViperIDE, and standalone applications package as AppImage, RPM, DMG, and Windows installers â€” and the native linker resolves cleanly on both Linux and Windows/MSVC toolchains, now rejecting imports exclusive to another platform. A closing hardening pass clamps every D3D11 shader-facing value before upload, lets ViperIDE's terminal, debugger, build, and source-control panels degrade cleanly under a flood of child output, and makes `Viper.Crypto` validate every input up front â€” including a ChaCha20-Poly1305 fix that restores interoperable AEAD tags.

- **The public API settles on one canonical name.** Aliases and duplicates are gone and the generator refuses to mint new ones â€” collection sizes read as `Count`, semantic lengths as `Length`, boolean probes return true/false, and `Entity3D` positions are plain properties, with both languages, ViperIDE, the demos, and the docs rebound to the smaller surface.
- **Recoverable failures come back as values.** Terminal reads, diagnostics, collection and channel pops, decryption, HTTP/REST/SMTP sends, data-format parsing, searches, shell commands, and game and 3D queries all gain `Option`- and `Result`-returning forms, so absence and error are values instead of nulls, sentinels, or side channels; the older forms remain for compatibility.
- **Terse names spell themselves out, and the sharp edges are labelled.** `LeadZ` becomes `CountLeadingZeros`, `NumSci` becomes `Scientific`, and factories drop redundant `New` prefixes, while manual memory, trap-state mutation, legacy ciphers, and the TLS-verification bypass move into plainly named `Runtime.Unsafe`, `Crypto.Legacy`, and testing-only homes so nothing dangerous hides behind an innocent name.
- **A machine-readable API contract (ADR 0027).** `viper --dump-runtime-api` keeps its shape and adds parsed types, ownership, stability, capability, and security metadata for every entry, while a new internal-only marker keeps implementation helpers out of the public dump.
- **Crypto input validation, and a ChaCha20-Poly1305 interop fix.** The AEAD path clamps its Poly1305 key with the RFC 8439 26-bit limb masks â€” it previously produced non-interoperable tags for some one-time keys â€” and every `Viper.Crypto` entry point validates its handles before use and halts after a validation trap instead of continuing with sentinel buffers.

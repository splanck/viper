# Viper Compiler Platform — Release Notes

> **Development Status:** Pre-Alpha. Viper is under active development and not ready for production use.

## Version 0.2.99 — Pre-Alpha (DRAFT — unreleased)

<!-- DRAFT: release date TBD. v0.2.7 was cut on 2026-06-30. -->

### What this release is about

A short cleanup-and-packaging follow-up to the v0.2.7 hardening cycle. The pre-alpha runtime surface finishes settling on its canonical public names — aliases and duplicates gone, the generator refusing to mint new ones, `Count`/`Length` and boolean shapes normalized — while packaging graduates from a toolchain-only path into a full release story that ships ViperIDE and packages standalone applications as AppImage, RPM, DMG, and Windows installers. A Linux native-build stability fix and a round of Game3D runtime surface round it out; line counts move only slightly because the API work is a rename-and-prune.

- **Runtime API surface canonicalized.** The public runtime API settles on one canonical name set: `RT_ALIAS` becomes a generator error, duplicate and alias entries are removed, cardinality splits into `Count` (collections/containers, plus `BitSet` population count) versus semantic `Length`, boolean probes and toggle setters use `i1`, acronyms move to PascalCase (`UseCcd`/`PostFx`/`Pbr`/`EntityA`/`EntityB`), and simple `Entity3D` setters give way to property assignment — with `--dump-runtime-api` now printing public signature text and the whole tree (both frontends, ViperIDE, demos, docs, golden IL) rebound to the smaller surface.
- **Implementation targets stay private.** A new `RT_INTERNAL_FUNC` / `publicSurface` descriptor bit keeps targets like `Random.inst_*` available to class-method lowering while dropping them out of generated externs and the API dump; concrete GUI widget handles inherit `Viper.GUI.Widget` methods without copying them into every class catalog.
- **Runtime API dumps gain production contract metadata (ADR 0027).** `viper --dump-runtime-api` keeps its existing names/signatures/classes shape and adds schema v2 fields for the `runtime-def-v1` signature dialect, parsed return/parameter types, row kind, owner, class kind, static-method classification, stability, capabilities, fallibility, ownership, and documentation anchor hints.
- **Terminal input gains explicit Option and Result APIs (ADR 0028).** `Viper.Terminal.TryReadLine`/`TryAsk` return `Viper.Option`, `ReadLineResult`/`AskResult` return `Viper.Result`, and the existing nullable `ReadLine`, `Ask`, and `InputLine` APIs stay available for compatibility.
- **Trap diagnostics gain a read-only snapshot API (ADR 0029).** `Viper.Diagnostics.CurrentTrap` returns `Option<TrapInfo>` with kind, code, instruction pointer, source line, message, kind name, and formatted location while preserving the existing `Viper.Error.*` compatibility hooks.
- **Runtime memory controls gain clearer namespaces (ADR 0030).** `Viper.Runtime.Unsafe` now owns manual retain/release helpers and `Viper.Runtime.GC` owns cycle-collector diagnostics, while existing `Viper.Memory` and `Viper.Memory.GC` names remain available for compatibility.
- **Core parse double helpers gain clearer names (ADR 0031).** `Viper.Core.Parse.TryDouble` and `DoubleOr` are the canonical double-parse helpers, while `TryNum` and `NumOr` remain compatibility aliases.
- **Math bit helpers gain full-name aliases (ADR 0032).** `CountLeadingZeros`, `CountTrailingZeros`, `RotateLeft`, `RotateRight`, and `ShiftRightLogical` are now the canonical readable names for the terse `LeadZ`, `TrailZ`, `Rotl`, `Rotr`, and `Ushr` helpers.
- **Core conversion formatting drops underscore-style canonical names (ADR 0033).** `ToStringInt` and `ToStringDouble` are the canonical string-formatting helpers, while `ToString_Int` and `ToString_Double` remain compatibility aliases.
- **Capacity naming is normalized (ADR 0034).** `Capacity` properties replace user-facing `Cap` in collection and channel docs, and `BinaryBuffer.NewCapacity` replaces `NewCap`; the old names remain compatibility aliases.
- **BloomFilter false-positive metrics are explicit (ADR 0035).** `FalsePositiveRate()` is now the canonical method name for the estimated rate, while `Fpr()` remains available for compatibility.
- **Formatting and frame-clamp abbreviations gain readable aliases (ADR 0036).** `Scientific`, `Percent`, `YesNo`, and `SetMaxDeltaTime` are the canonical names for the old `NumSci`, `NumPct`, `BoolYN`, and `SetDTMax` helpers.
- **Collection write verbs are clearer (ADR 0037).** `LruCache.Set`, `BiMap.Set`, and `MultiMap.Add` are the canonical collection write names, while existing collection `Put` aliases remain available; HTTP `Put` keeps its protocol meaning.
- **Graphics named factories drop redundant `New` prefixes (ADR 0038).** Mesh, material, light, collider, and Game3D world factories now have canonical names such as `Mesh3D.Box`, `Material3D.PBR`, `Light3D.Point`, `Collider3D.Capsule`, and `World3D.WithCamera`; old `New*` names remain aliases.
- **Value-producing failure helpers gain Option aliases (ADR 0039).** Collections, channels, futures, DateTime parsing, locale parsing, and message lookup now expose explicit `*Option` helpers such as `Queue.TryPopOption`, `Channel.TryRecvOption`, `Future.TryGetOption`, `DateTime.TryParseOption`, `Locale.TryParseOption`, and `MessageBundle.TryGetOption`; legacy null, empty-string, and numeric-sentinel APIs remain available.
- **Keyboard constants move to a canonical input namespace (ADR 0040).** `Viper.Input.Key` now owns readable key-code constants such as `Key.A`, `Key.Digit1`, `Key.LeftShift`, and `Key.NumpadDecimal`; existing `Keyboard.Key*` and `Game3D.Keys` constants remain compatibility mirrors.
- **Crypto decryption gains production failure shapes and legacy namespaces (ADR 0041).** `Viper.Crypto.Cipher` and `Viper.Crypto.Aes` now expose Result-returning and Option-returning decrypt APIs so authentication failures, malformed frames, invalid keys, and traps are explicit values, while AES-CBC and legacy hash algorithms move into `Viper.Crypto.Legacy.Aes` and `Viper.Crypto.Legacy.Hash` compatibility namespaces.
- **HTTP TLS verification bypasses are explicitly testing-only (ADR 0042).** `Viper.Network.HttpReq.AllowInsecureCertificatesForTesting()` exposes the existing certificate-verification bypass under a name that is hard to miss in reviews and generated API dumps; `SetTlsVerify` remains available for compatibility and is marked unsafe.
- **Random chance now reads as a predicate (ADR 0043).** `Viper.Math.Random.Chance(probability)` now returns `Boolean`; the old 0/1 integer shape remains available as `ChanceInt(probability)` for compatibility.
- **Crypto module policy toggles expose process scope (ADR 0044).** `Viper.Crypto.Module.EnableApprovedModeForProcess`, `DisableApprovedModeForProcess`, and `IsApprovedModeForProcess` make approved-mode state explicitly process-wide; the old shorter names remain compatibility rows, and mutating policy toggles are marked unsafe in API metadata.
- **Boxed value-type runtime hooks move under Runtime.Unsafe (ADR 0045).** `Viper.Runtime.Unsafe.ValueType` and `ValueTypeAddField` are the canonical names for compiler/runtime boxed-struct interop; `Viper.Core.Box.ValueType`, `ValueTypeAddField`, and `Viper.Core.ValueType.AddField` remain compatibility rows.
- **Game3D reusable model loading adopts prefab terminology (ADR 0046).** `Viper.Game3D.Prefab.Load`, `LoadAsset`, `LoadAsync`, and `LoadAssetAsync` are the canonical names for loading cached reusable prefab templates, with `AssetHandle3D.GetPrefab` for async results; `Assets3D.LoadTemplate*` and `GetTemplate` remain compatibility rows.
- **HTTP requests gain explicit Result-returning APIs (ADR 0047).** `HttpReq.SendResult` and `RestClient.GetResult`/`PostResult`/`PutResult`/`PatchResult`/`DeleteResult`/`HeadResult` return `Result<HttpRes>` so transport failures are values and HTTP statuses remain inspectable on the response; `RestClient.LastStatus`, `LastResponse`, and `LastOk` remain compatibility diagnostics.
- **Trap-state mutation hooks move under Runtime.Unsafe (ADR 0048).** `Viper.Runtime.Unsafe.SetThrowMsg`, `ClearThrowMsg`, `SetTrapFields`, and `RaiseKind` are the canonical names for generated-code/runtime trap-state mutation, while `Viper.Diagnostics.CurrentTrap` remains the read-only application API and the old `Viper.Error.*` mutation names remain compatibility rows.
- **Terminal compatibility signatures drop nullable suffix text (ADR 0049).** `Viper.Terminal.ReadLine`, `Ask`, and `InputLine` still preserve their existing EOF-compatible behavior, but public runtime metadata now prints ordinary `str` signatures and points callers to `TryReadLine`/`TryAsk` or the Result forms for explicit EOF handling.
- **Game query/path/event APIs gain snapshot result objects (ADR 0050).** `Pathfinder.FindPathResult`, `FindNearestResult`, `Quadtree.QueryRectResult`, `QueryPointResult`, `QueryPairs`, `AnimStateMachine.PollEvents`, and `AnimTimeline.PollEvents` return stable result or event-batch values, while existing `Last*`, `ResultCount`/`GetResult`, pair-index, and event-index APIs remain compatibility diagnostics.
- **PathResult exposes an explicit step count (ADR 0062).** `PathResult.StepCount` is the canonical cell-to-cell step count; `PathResult.Length` stays available as a compatibility alias with migration metadata.
- **Data format parsers gain Result-returning APIs (ADR 0051).** `JsonStream.NextResult`, `Xml.ParseResult`, `Yaml.ParseResult`, `Serialize.ParseResult`, and `Serialize.AutoParseResult` keep parse failures attached to the returned value, distinguish valid YAML/null documents from failures, and leave the existing `Next`/`Parse`/`AutoParse` plus `Error` side-channel APIs available as compatibility diagnostics.
- **Connect/open/load setup APIs gain Result-returning variants (ADR 0052).** `Tls.ConnectResult`/`ConnectForResult`/`ConnectOptionsResult`, `Pty.OpenResult`, and `SceneDocument.LoadResult`/`LoadJsonResult` return `Ok(handle)` or `Err(message)` for routine setup and user-input failures, while the existing `Error`, `LastError`, and diagnostic-document paths remain available for compatibility and tooling.
- **Search helpers gain Option-returning APIs (ADR 0053).** Collection, XML, regex, 2D scene, SceneDocument, audio group, and LazySeq search helpers now expose `FindOption`/`FindPosOption` variants so absence is `None`, found indexes are `SomeI64(index)`, and regex empty-string matches are no longer ambiguous; old `-1`, `NULL`, and empty-string search APIs remain compatibility rows.
- **UnionFind root lookup gains an Option-returning API (ADR 0054).** `UnionFind.FindRootOption(x)` returns `Some(root)` for valid element indexes and `None` for invalid input, while the conventional `Find(x)` method remains available as the legacy `-1` sentinel form.
- **Shell command capture gains a side-channel-free result object (ADR 0055).** `Exec.ShellResult(command)` returns a `Viper.System.CommandResult` with `Output`, `ExitCode`, and `Succeeded`, replacing the fragile `ShellFull()` plus `LastExitCode()` pairing for new code.
- **Game UI table clicks now have a structured result object (ADR 0056).** `Table.HandleClickResult(x, y)` returns `Viper.Game.UI.TableClickResult` with row/header booleans and Option-returning row/column accessors, while `HandleClick()` and `LastHeaderClick()` remain as compatibility APIs.
- **SMTP sends gain Result-returning APIs (ADR 0057).** `SmtpClient.SendResult(...)` and `SendHtmlResult(...)` return `OkI64(1)` or `ErrStr(message)`, replacing the `Send()`/`SendHtml()` plus `LastError` side-channel for new network code.
- **GUI lookup helpers gain Option-returning variants (ADR 0058).** `FindBar.FindNextOption`/`FindPrevOption`, `TestHarness.FindBy*Option`, and `CommandRegistry.FindOption` make preview GUI absence handling consistent while preserving existing Boolean, map-sentinel, and NULL-returning methods.
- **Graphics3D/Game3D lookups gain Option-returning variants (ADR 0059).** `SceneGraph.FindOption`, `SceneNode.FindOption`, `SceneAsset.FindNodeOption`, `World3D.FindNodeOption`, `World3D.FindEntityOption`, `Skeleton3D.FindBoneOption`, and `NavMesh3D.FindPathOption` replace null/`-1` checks for new 3D code while preserving the existing lookup methods.
- **SceneAsset loaders gain Result-returning variants (ADR 0060).** `SceneAsset.LoadResult`, `LoadAssetResult`, `LoadAnimationResult`, `LoadAnimationAssetResult`, `LoadNodeAnimationResult`, and `LoadNodeAnimationAssetResult` return `Ok(value)` or `Err(message)` for routine content and clip failures; the existing `Load*` methods and `AssetDiagnostics3D.LastLoadError*` remain compatibility diagnostics.
- **Zia semantic jobs expose optional error text (ADR 0061).** `SemanticJob.ErrorOption(job)` returns `SomeStr(message)` when a background language-service job failed and `None` otherwise, replacing empty-string checks on `SemanticJob.Error(job)` for new IDE code.
- **`Entity3D` position accessors become properties (ADR 0026).** `Position`/`WorldPosition` reclassify from zero-argument methods to read-only properties, and Zia sema now resolves runtime-class member access up front — emitting `V-ZIA-METHOD-CALL` with an add-`()` fix-it when a zero-arg method is read as a field, and an internal compiler error instead of a mistyped fallback for a genuinely unknown member.
- **Toolchain installers now ship ViperIDE.** The installable toolchain builds and stages ViperIDE by default (`VIPER_INSTALL_VIPERIDE`, generated `viperide.buildinfo`), installer manifests require every user-facing binary, and Linux/macOS/Windows packages gain desktop launchers, command symlinks, branded dependency-free Viper icons, and payload-membership verification.
- **Applications package on all three platforms (ADR 0025).** Standalone apps now ship as first-class AppImage, RPM, DMG, and Windows-installer targets with AppStream metadata, RPM `Requires`, LaunchServices roles, and publisher/wizard metadata; a manual Windows Release Installer workflow builds through the canonical script, restores optional PFX signing, verifies, and uploads the installer; and the package CLI gains license/readme/welcome and platform-specific options, inline `--option=value` forms, a `macos-dmg` alias, and `viper package --dry-run --json`.
- **Linux native build stability restored.** Outlined AArch64 atomics are disabled so static archives stop importing libgcc helpers the native linker can't resolve, `__once_proxy` is recognized as a libstdc++ runtime import, signature-whitelist registration drops `std::call_once` for TLS-relocation-free function-local statics, and non-interactive installs fall back from `/usr/local` to `build/install` when no prefix is set.
- **Game3D runtime surface and graphics test knobs.** `World3D.SpawnHeightfieldCollider` lets a standalone `Terrain3D` join physics and character-controller collision, `Water3D.SetPosition` centers planes over off-origin terrain, `Viper.Game3D.Keys` mirrors the full shared keyboard table, `Canvas3D.BackendSupports` gains CPU/GPU post-FX aliases, and the split GPU post-FX path replays the final overlay so HUD composites into presented and captured frames; `VIPER_GFX_NO_ACTIVATE`/`VIPER_GFX_HIDE_WINDOWS` (macOS and Linux/X11) let display-requiring CTest cases render without stealing focus.

Demos and docs tracked the work: the `game3d-showcase` was rebuilt around the new runtime — a `CharacterController3D` player on a spawned terrain heightfield with first-person/free-look, sprint stamina, scanner-gated beacons, boulder physics, nav-mesh critters, and weather, over split-out movement/camera/math modules — while the packaging, Game3D, CLI, and tools docs and man pages absorbed the new installer payloads, options, and canonical API names.

Two Languages, One IL

Viper ships two complete programming languages that compile through the same typed, SSA-based intermediate language. This proves the IL architecture works as a genuine abstraction layer — not just an implementation detail.

Zia — The Modern Language

Zia is a statically-typed, modern language designed for building real applications:

  • Classes (reference-type) with inheritance, interfaces, destructors, and `final` enforcement
  • Structs (value-type, stack-allocated) with copy semantics
  • Generics with type constraints
  • Pattern matching with exhaustiveness checking (including enum variants)
  • Enums with discriminant values and exhaustiveness checking in `match`
  • Optional types with safe navigation (?.) and unwrap (!)
  • Lambdas and single-expression functions ((x: Integer) => x * 2, func f(x) -> T = expr;)
  • Variadic parameters (func sum(nums: ...Integer))
  • Type aliases (type Name = TargetType;)
  • Shift & compound bitwise operators (<<, >>, <<=, >>=, &=, |=, ^=)
  • Polymorphic is — subclass-aware runtime type checks
  • String interpolation ("${name}")
  • Module system with bind imports
  • Try/catch exception handling with typed catch (e.g. catch(e: DivideByZero))
module GameDemo;

bind Viper.Terminal;
bind Canvas = Viper.Canvas;

class Player {
    hide Integer x;
    hide Integer y;
    hide Integer health;

    expose func init(startX: Integer, startY: Integer) {
        x = startX;
        y = startY;
        health = 100;
    }

    expose func update(dx: Integer, dy: Integer) {
        x = x + dx;
        y = y + dy;
    }

    expose func draw(c: Canvas) {
        c.Box(x, y, 16, 16, 0x00FF00);
    }

    expose func isAlive() -> Boolean {
        return health > 0;
    }
}

BASIC — The Classic Language

Viper BASIC modernizes the classic language with OOP features while keeping its familiar syntax:

  • Classes with inheritance, interfaces, and constructors
  • Namespaces with USING imports
  • Enums (ENUM...END ENUM)
  • SELECT CASE for multi-way branching
  • Dynamic arrays with runtime resizing
  • String operations (LEN, LEFT$, MID$, RIGHT$, UPPER$, LOWER$)
  • Subroutines and functions with local scope
CLASS Player
    DIM x AS INTEGER
    DIM y AS INTEGER
    DIM health AS INTEGER

    SUB Init(startX AS INTEGER, startY AS INTEGER)
        x = startX
        y = startY
        health = 100
    END SUB

    SUB Update(dx AS INTEGER, dy AS INTEGER)
        x = x + dx
        y = y + dy
    END SUB

    FUNCTION IsAlive() AS BOOLEAN
        IsAlive = (health > 0)
    END FUNCTION

    SUB Draw()
        Viper.Terminal.SetPosition(y, x)
        IF health > 0 THEN
            PRINT "@";
        END IF
    END SUB
END CLASS

The Shared IL

Both languages compile to the same Viper IL — a typed, SSA-inspired intermediate language. This means:

  • Both languages produce identical runtime behavior for equivalent programs
  • The same optimizer passes apply to both
  • Both can compile to the same 6 native targets
  • Zia and BASIC modules can be linked together via IL linking
il 0.2.0
func @Player.isAlive(ptr %self) -> i1 {
entry(%self:ptr):
  %hp_ptr = gep %self, 16
  %hp = load i64, %hp_ptr
  %alive = scmp_gt %hp, 0
  ret %alive
}

The IL above is what both the Zia and BASIC versions of isAlive() compile to — identical output from different source syntaxes.

Build Your Own Frontend

The IL is a stable, documented interface. You can write your own language frontend that targets Viper IL and immediately get access to the optimizer, VM, native codegen, and entire 300-class runtime. The architecture documentation and the existing frontends serve as reference implementations.

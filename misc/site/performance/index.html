Performance

Viper native code benchmarked against C, Rust, Java, C#, Lua, and Python on identical algorithms. All benchmarks run on Apple M4 Max with 10 iterations and 2 warmup runs.

Headline Results

  • 1.8x faster than C -O2 on branch-heavy code (branch_stress)
  • 2.2x faster than Rust -O on division workloads (udiv_stress)
  • 8 of 9 benchmarks faster than C -O0
  • 50-160x faster than Python across all benchmarks

Methodology

  • Hardware: Apple M4 Max
  • Iterations: 10 per benchmark, 2 warmup
  • Fairness: Identical algorithms across all languages, verified return values match
  • Benchmarks: 9 stress tests covering arithmetic, branching, function calls, division, inlining, string ops, and redundancy elimination

Why Viper Wins on Some Benchmarks

  • branch_stress: Viper's strength reduction turns division/modulo into cheaper operations; Clang's auto-vectorization doesn't help on branchy code
  • udiv_stress: Custom SMULH magic-number multiply for constant divisors (4-6 cycles vs 22 cycles for SDIV)
  • call_stress: Tight calling convention with effective inlining; faster than C -O0's unoptimized call overhead

Where Viper Has Room to Grow

  • arith_stress: Clang's loop vectorization (SIMD) gives C -O2 a ~5x advantage on pure arithmetic
  • fib_stress: Clang converts one recursive branch to a loop; Viper does naive double recursion
  • redundant_stress: Clang's aggressive dead code elimination at -O2 removes more redundant work

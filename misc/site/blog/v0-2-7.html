v0.2.7: Graphics3D Stability, a ZannaIDE Workbench, and Fail-Closed Hardening

v0.2.7 June 2026

A hardening cycle continuing v0.2.6, with focused new capability in 3D, tooling, and the languages. The 3D renderer stops flickering frame-to-frame and gains spot-light shadows, a deterministic multi-threaded software rasterizer, and node animation over a rebuilt glTF/FBX importer; ZannaIDE becomes a real workbench with a VM debugger, git source control, multi-root workspaces, and an integrated terminal; and the runtime's hardening backbone makes bad input fail closed — returning a safe value instead of crashing your program. The full release notes are at docs/release_notes/Zanna_Release_Notes_0_2_7.md; this is the highlight reel.

Graphics3D & rendering

  • Flicker, fixed — queue-order-independent occlusion history, camera-depth-fitted shadow cascades, per-camera LOD/impostor hysteresis, and terrain-horizon culling end per-frame visible-triangle flicker; every backend validates draw-command index ranges.
  • Spot-light shadow maps — new — shadow-casting spot lights join the shared directional shadow budget across all four backends (software/OpenGL/Metal/D3D11).
  • Deterministic software rasterizer — a worker-pool rasterizer reproduces the single-threaded image bit-for-bit, so headless renders stay reproducible while throughput scales with cores.
  • Open-world streaming — new — WorldStream3D cells load into a resident-byte budget and NavMesh3D.Export/Import round-trip versioned navmesh assets.

3D assets & animation — new

  • Node animation — NodeAnimation3D/NodeAnimator3D play node, camera, and morph-weight clips over a rebuilt FBX importer (full transforms, cubic curves, PBR routing, async loading).
  • Loaders fail closed — corrupt, truncated, or oversized images and models return null with queryable diagnostics; a shared count guard and libFuzzer harnesses keep the parsers crash-free.
  • Degradation is observable — a process-wide Game3D.Diagnostics3D surface counts the rare correct-but-degraded fallbacks so a smoke run can assert a clean frame.

ZannaIDE — a real workbench

  • VM debugger — zanna run --debug-adapter drives pause/continue, step in/over/out, breakpoints, call stacks, and locals, and evaluates a watch expression or the identifier under the cursor against the stopped frame.
  • Editor — Zanna/IL syntax highlighting with a compiler-driven semantic overlay, indentation guides, right-click context menus, and two-language formatting, plus BASIC completion, diagnostics, hover, and symbols.
  • Workbench — an activity bar with a full git Source Control view, multi-root workspaces, and an integrated terminal over a new Zanna.System.Pty pseudo-terminal runtime (ConPTY on Windows, posix_openpt/fork on POSIX).

Agent-facing CLI — new

  • Machine-readable diagnostics — zanna check/eval/explain over a central diagnostic-code catalog, plus --dump-runtime-api/--dump-opcodes registry inventories; the LSP/MCP servers carry the same structured diagnostics.

Runtime fails closed — repo-wide

  • Bad input degrades, not crashes — every recoverable runtime error returns a safe value instead of trapping; the lone exception, unrecoverable entropy/DRBG failure, aborts outright so no path proceeds with predictable key material.
  • New surfaces — packed F64Buffer/I64Buffer, a poll-based System.Shutdown, embedded-IANA Time.TimeZone, and low-pass/high-pass/EQ/delay/reverb audio insert chains.
  • Parsers & crypto — CSV/JSON/HTML/XML/Markdown/TOML parsers fail closed and reach further, and RSA verification goes constant-time.

VM, codegen & languages

  • One program, one result — a VM-neutral kernel makes the tree-walking VM and both bytecode engines yield identical values and trap kinds, and threaded Parallel/Pool callbacks now run under the VMs.
  • Codegen — a performance round plus two optimizer fixes retire the last -O0 demo carve-outs, so every demo builds and runs at -O2; a VM↔native determinism gate closed four divergences.
  • Frontends — BASIC arrays move to inclusive upper bounds with overflow-checked indexing, and Zia gains Map[Integer, T] and header-optional modules.

2D drawing — new

  • Text & gradients — Pixels gains bitmap-font text drawing and measurement, and Gradient2D adds normalized Sample/SampleRGBA — the runtime behind a rebuilt Zanna Paint with layers, blend modes, and an HSL picker.

Runtime class names disambiguated

  • Unique leaf names — colliding Zanna.* classes are renamed unique (Graphics.Scene→SceneGraph, Sound3D→SpatialAudio3D, the Game.UI widgets that shadowed GUI ones→Hud*), with the old names kept working as aliases.

By the numbers

  • Production SLOC: ~669K → ~762K (+93K)
  • Source files: 3,096 → 3,402 (+306)
  • Test SLOC: ~278K → ~304K (+26K)
  • Commits: 191 since v0.2.6

Demos shipped

The game3d-showcase gained F11 fullscreen, wind-swayed foliage, a render-to-image minimap, spot shadows, and a renderer/audio counter HUD; the paint app was rebuilt around a registry-driven tool set with layers, blend modes, and an HSL color picker; and crackman gained a vector chomp-and-death animation, pulsing power pellets, score popups, and a board-clear celebration.

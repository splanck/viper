v0.2.4: Game Engine, Native Linker, 3D Backends, Toolchain Packaging

v0.2.4 April 2026 · DRAFT

Viper's largest release to date. The native toolchain becomes self-sufficient on every platform, the 3D graphics stack reaches feature parity across Metal / D3D11 / OpenGL / software, the runtime grows to 300 classes across 22 modules, and Zia gains seven new language features. The full release notes live in the repo at docs/release_notes/Viper_Release_Notes_0_2_4.md; this is the highlight reel.

Native toolchain — zero external dependencies on every platform

  • Native PE/COFF linker pipeline — Full Windows native linking without clang or link.exe. ArchiveReader, SymbolResolver, SectionMerger, DeadStrip, ICF, RelocApplier, and PeExeWriter with proper .idata import tables. viper build now produces native .exe end-to-end with zero external tool dependencies.
  • ELF dynamic linking — ElfExeWriter rewritten from static-only to full dynamic-link support: PT_INTERP, PT_DYNAMIC, .dynsym, .dynstr, .hash (SYSV), .rela.dyn with R_X86_64_GLOB_DAT. Linux x86_64 programs natively link against libc/libm/libpthread/libX11/libasound without falling back to the system linker.
  • Mach-O hardening — ObjC metadata sections emit proper section types (S_CSTRING_LITERALS, S_LITERAL_POINTERS, S_ATTR_NO_DEAD_STRIP, S_COALESCED), framework auto-linking via symbol normalization, ad-hoc code signing, and a symbol-index sentinel fix in MachOWriter so symbols resolved to slot 0 are no longer dropped.
  • Native Exception Handling lowering — New NativeEHLowering pass (683 LOC) rewrites structured IL EH markers into ordinary calls and branches before backend lowering, enabling cross-platform native exception handling on x86_64 and AArch64.
  • Toolchain packaging foundation — New viper install-package CLI command with ToolchainInstallManifest as the shared data model; Windows / macOS / Linux package builders gain toolchain entry points.
  • Installed-layout discovery — Layered runtime archive search: VIPER_LIB_PATH env → exe-relative ../lib → platform standard paths → build-tree fallback. An installed Viper can compile native executables without a build tree.

3D graphics backends — Metal and D3D11 feature-complete

  • Metal — All 14 backend plans implemented (47% → 94% feature parity with the software renderer). GPU skinning, morph targets (with normal deltas), shadow mapping, terrain splatting, post-processing, instanced rendering, fullscreen-triangle skybox, direct vs GPU-postfx window paths, cached RTT and morph payloads.
  • D3D11 — All 20 backend plans implemented in a 3,173-line HLSL+C backend rewrite. Full PBR with Cook-Torrance + GGX, GPU skinning, morph targets, shadow mapping, instanced rendering, terrain splatting, advanced post-FX (DOF, motion blur, SSAO, chromatic aberration), cubemap skybox, environment reflections.
  • Software renderer upgrades — Per-pixel terrain splatting (4-layer weight blend), bilinear filtering, vertex color, shadow mapping, material shader hooks (Toon / Fresnel / Emissive shading models).
  • Generation-aware texture caching — All GPU backends now track a generation counter on Pixels and CubeMap3D objects. In-place mutations re-upload textures without allocating new GPU resources.

Runtime library — 300 classes, 22 modules

  • 3D graphics suite — PBR materials (Material3D.NewPBR), Collider3D with 7 shape types (box, sphere, capsule, hull, mesh, heightfield, compound), Body3D/Physics3D with quaternion orientation, sleep, CCD, joints, world queries (raycast / sweep / overlap), collision events with contact manifolds, NavAgent3D A* pathfinding, AudioListener3D/AudioSource3D for spatial audio, Model3D unified asset import (.vscn/.fbx/.gltf/.glb), AnimController3D state machine.
  • Media codecs — From-scratch JPEG, GIF (animated), OGG Vorbis, MP3 decoders. PNG decoder extended to all color types, bit depths, interlacing, transparency. WAV loader extended to 24-bit and float32 PCM. OGG/MP3 music streaming with on-the-fly resampling.
  • Video playback — VideoPlayer runtime class with AVI/MJPEG decoder (DHT injection), Theora codec infrastructure, OGG multi-stream demux, GUI VideoWidget.
  • Game engine APIs — 10 new systems: Entity, Behavior, Raycast2D, LevelData, SceneManager, Camera.SmoothFollow, AnimStateMachine named states, MenuList.HandleInput, Config.Load, Tilemap.SetTileAnim. Plus PlatformerController, Lighting2D, AchievementTracker, Typewriter.
  • Asset embedding — Compile-time VPA packaging via embed / pack / pack-compressed project directives. Assets injected into .rodata for zero-file-dependency executables.
  • HttpServer runtime class wired through both Zia and BASIC frontends with Listen, Accept, Respond, Close, and request property accessors.
  • Native macOS menu bar — 554-line ObjC bridge mirrors Viper GUI menubars to NSMenu with About / Preferences / Quit relocation, Cmd+key accelerators, Services submenu.
  • GUI overhaul — Hover events (MOUSE_ENTER/LEAVE) tracked in toolkit runtime state, two-pass painting (normal + overlay) so dropdowns/popups/floating panels render above siblings, justify-content layout (CENTER/SPACE_BETWEEN/SPACE_AROUND/SPACE_EVENLY), ScrollView clipping, FloatingPanel children promoted to real widget tree, per-app theme ownership.

Zia language additions

  • Variadic parameters: func sum(nums: ...Integer)
  • Type aliases: type Name = TargetType;
  • Shift operators (<<, >>) and compound bitwise assignments (<<=, >>=, &=, |=, ^=)
  • Single-expression functions: func f(x: Integer) -> Integer = x * 2;
  • Lambdas: (x: Integer) => x * 2
  • Polymorphic is expressions — subclass-aware runtime type checks
  • Enum support in both Zia and BASIC, with exhaustiveness checking in match
  • final enforcement — semantic analyzer rejects reassignment of final variables, for-in loop variables, and match pattern bindings
  • Keyword rename: entity → class, value → struct across all source, tests, REPL, LSP, docs, and the VS Code extension.
  • Completion runtime APIs — Viper.Zia.Completion.Check / .Hover / .Symbols expose the compiler's analysis pipeline directly to Zia programs (used by ViperIDE).

Codegen & optimizer

  • Codegen pipeline decomposition — Both x86_64 and AArch64 backends refactored from monolithic per-function pipelines into composable PassManager-based pass architectures.
  • AArch64 join-phi coalescing — New peephole pass eliminates stack round-trips at CFG join blocks by replacing them with topologically-ordered register moves.
  • x86-64 carry-through — canCarryIntoNextBlock() avoids forced spills at single-predecessor fallthrough edges. Branch relaxation (short JMP / Jcc) reduces code size.
  • Register allocator hardening — AArch64 protected-use eviction, isLiveOut() query, block-epoch spill slot lifetimes, FPR load/store classification.
  • IL optimizer correctness — EarlyCSE and GVN now enforce textual block ordering when replacing across dominator-ordered blocks. SCCP declares preserved analyses (dominators, loop info, CFG).

Cross-platform infrastructure

  • PlatformCapabilities.hpp with VIPER_HOST_* / VIPER_CAN_* macros replacing scattered raw _WIN32/__APPLE__/__linux__ checks.
  • CMake capability gate (VIPER_GRAPHICS_MODE/VIPER_AUDIO_MODE) with AUTO/REQUIRE/OFF modes.
  • Generated RuntimeComponentManifest for machine-checked archive names — drift between runtime.def and linker discovery is now a build error.
  • Platform import planners split from monolithic NativeLinker.cpp into MacImportPlanner.cpp, LinuxImportPlanner.cpp, WindowsImportPlanner.cpp.
  • Unified build_viper_unix.sh replacing near-identical mac/linux scripts.
  • Platform policy lint script lint_platform_policy.sh flags raw _WIN32/__APPLE__/__linux__ usage outside approved adapter files.

By the numbers

  • Production SLOC: ~348K → ~446K (+98K)
  • Source files: 2,671 → 2,869 (+198)
  • Test count: 1,351 → 1,454 (+103)
  • Runtime classes: 229 → 300 across 22 modules

Demos shipped

XENOSCAPE Metroid-style sidescroller (~17K LOC), 3D bowling with Physics3D pin collision, 3D world starter demo, ViperSQL grown to ~78K LOC, ViperIDE with live diagnostics + IntelliSense + project search, plus Chess, Pac-Man, and 8 Graphics3D API demos.

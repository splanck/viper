v0.2.6: Game3D Engine, ZannaIDE Editor, Pointer-Safety Gate, Deep Hardening

v0.2.6 June 2026

A hardening cycle pushing toward alpha quality that also delivered substantial new capability. The headline work is a code-first Zanna.Game3D engine layered over the 3D runtime and a real ZannaIDE editor with build/run and language services. Underneath, a hardening backbone swept memory, threads, crypto, I/O, graphics, the bytecode VM, the native toolchain, and packaging, while the Zia frontend closed its major language gaps and raw pointers left both source languages. The full release notes are at docs/release_notes/Zanna_Release_Notes_0_2_6.md; this is the highlight reel.

Code-first Game3D engine — new

  • One loop — World3D drives scene, physics, camera, input, audio, and effects from a single fixed-step update/render loop with an internal worker, floating-origin rebasing, and streaming.
  • Entities — Entity3D gives a transform/mesh/body hierarchy with cycle-safe, ownership-checked parenting and recursive mesh/material propagation.
  • Built-ins — first-person, free-fly, orbit, and follow cameras, a grounded character controller, Animator3D, spatial Sound3D, and Effects3D VFX presets.
  • Tested vertical slice — examples/3d/openworld_slice/ streams terrain and cells with heightfield colliders, a skinned glTF agent with crossfade and foot IK, and named macOS and Windows perf baselines.

ZannaIDE editor — real build/run and language services

  • Build/run loop — now a standalone top-level project with an argument-vector build/run loop, streamed cancellable output, clickable diagnostics, gutter breakpoints, Quick Open, and workspace symbols.
  • Live language services — zia force-loads fe_zia, so completion, diagnostics, hover, signatures, outline, and call hierarchy hit the real semantic engine, content-revision-gated off the keystroke path.
  • Editor runtime — a streaming Zanna.System.Process, a structured Zanna.Zia.Toolchain, a semantic Zanna.Zia.ProjectIndex, and an editable Zanna.Game2D.SceneDocument document back the tree, completion, diagnostics, and symbols. (Debugging is still a non-executing placeholder.)
  • Link-time win — a codegen and dead-strip rewrite cut the ZannaIDE native x64 build from ~340s to ~35s.

Pointer-safety gate

  • The typed surface is the only surface — Zia and BASIC now reject raw Ptr types and pointer-signature runtime APIs. This is the biggest user-visible change.

Zia language stability

  • Control flow — defer; structured try/catch/finally, multi-catch, and bare rethrow; Result[T] with ? propagation.
  • Types — weak fields, function references, constrained generics, default interface methods, and declaration-order independence.

Hardening sweep

  • Memory, GC & threads — validated retain/release wrappers, weak-ref CAS retain inside the GC lock, trap-safe finalizers, class-ID validation on every public threads/MessageBus entry, and saturated wait deadlines.
  • Crypto, TLS & I/O — canonical Zanna.Crypto.* (scrypt, AES-GCM+AAD, fixed-schedule ECDSA P-256), TLS Key-Usage/Basic-Constraints/EKU enforcement, and hardened temp-file/archive/ZIP64 paths.
  • Network — independent HPACK tables, strict RFC 7230 Transfer-Encoding parsing (closes a request-smuggling avenue), and WebSocket frame/close-code validation.
  • Native toolchain — bounds, alignment, and reloc-correctness rounds across all four object readers and three writers; a strict runtime import audit reconciles the Release MSVC CRT surface with the import planner.

3D runtime & graphics

  • Asset pipeline — a GLTF class on an allocation-free, DOM-free JSON scanner with validated accessors; FBX embedded-texture decode and n-gon triangulation; multi-material OBJ/STL import into a SceneAsset node hierarchy; in-memory KTX2 with BC7/ETC2/ASTC software decode.
  • Correctness — corrected inverse-transpose normal matrices, bone-topology-frozen skeletal animation, and a multi-backend (software/OpenGL/Metal/D3D11) skinning, shadow, and readback round.
  • Physics3D — a warm-started sequential-impulse solver with sleep islands settles deterministic resting box stacks; shape-accurate SAT/GJK-EPA narrow phase; the implementation split into character/collision/query/solver units.

Standard-library namespace de-clutter — breaking

  • Seven root modules re-home under their documented taxonomy with no back-compat aliases: Lazy/LazySeq → Zanna.Functional, Machine/Environment/Exec → Zanna.System, Log → Zanna.Diagnostics, Fmt → Zanna.Text. Math, String, Terminal, and intrinsic Option/Result/Error stay at root.

Toolchain installers

  • Feature parity across platforms — native-emitted Windows .msi/.exe, macOS .pkg, and Linux .deb/.rpm/tarball packages with signing, file associations, dependency advertisement, and deep post-build verification. macOS generation no longer shells out to pkgbuild/productbuild.

By the numbers

  • Production SLOC: ~552K → ~669K (+117K)
  • Source files: 2,996 → 3,096 (+100)
  • Test SLOC: ~228K → ~278K (+50K)
  • Commits: 281 since v0.2.5

Demos shipped

The 3D Bowling demo became a playable arcade lane and migrated onto the Game3D layer alongside a new code-first Game3D sample set (a sub-20-line hello scene, a copyable starter, and a full-stack showcase), plus chess and Crackman polish.

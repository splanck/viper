v0.2.5: Runtime Hardening, HTTP/2, Localization, Compiler Throughput

v0.2.5 May 2026

A hardening-and-polish cycle with several notable new capabilities. Integer-overflow, handle-validation, and lifetime-correctness passes were applied across every runtime subsystem. The network stack became a full platform with TLS-backed servers and from-scratch HTTP/2. A new eleven-class Zanna.Localization.* namespace was added with zero external dependencies. Compiler throughput improved significantly through parallelism and a zero-copy pipeline. The full release notes are at docs/release_notes/Zanna_Release_Notes_0_2_5.md; this is the highlight reel.

Runtime hardening

  • Integer-overflow guards — Overflow-checked arithmetic, size-limit validation, and handle-type guards applied across graphics, text, threads, audio, I/O, and network subsystems.
  • Lifetime correctness — Timeout clamping, use-after-free fixes in audio playlist and crossfade paths, OGG page CRC-32 re-enabled, Ogg resync uses sliding-window scan.
  • Audio fixes — Master volume persisted before lazy backend init so settings round-trip without an audio device; rt_music_seek clamps to track duration; WAV header writer uses explicit LE helpers eliminating strict-aliasing UB; note sort is now stable.
  • IO hardening — IO handle type-safety system, File.MoveOver, directory/path hardening, LineReader EOF fix, SaveData UTF-8 validation.

2D graphics class expansion

  • ~40 new classes on top of Pixels / Canvas: offscreen surfaces, materials, shaders, post-effects, viewport scaling, tile layers, path drawing, text layout, nine-slice UI, animation, and a render-pass graph.
  • Wrong-class handle contracts — Every 2D class validates its handle type and returns a safe default rather than crashing on a misuse.

Full 3D asset pipeline

  • glTF and FBX import real skeletons, per-vertex skinning, and animations including sparse-accessor morph deltas and the full KHR extension suite.
  • STL loader hardening — NaN/inf vertices and out-of-range floats rejected in both binary and ASCII formats.
  • Handle-safety contracts — Every major 3D class (Mesh3D, SceneGraph, Canvas3D, Camera3D, Collider3D, Light3D, Material3D, Particles3D, Physics3D, Water3D) tested for NULL and wrong-type handle safety.

Network stack — now a platform

  • TLS servers — HttpsServer and WssServer backed by the in-tree TLS layer.
  • From-scratch HTTP/2 — HPACK header compression, stream multiplexing, flow control, server push.
  • Native RSA and X.509 — In-tree RSA implementation and X.509 certificate validator with no external dependencies.
  • Cookie jar, streaming downloads, connection pooling — Full HTTP client feature set.

Zanna.Localization.* — new

  • Eleven-class namespace: locale-aware number, date, time, and list formatting; translation catalogs; CLDR plural selection; text-direction utilities.
  • Zero external dependencies — all locale data compiled in-tree.

Structured diagnostics & tooling

  • Source-location snippets with caret underlines in compiler output.
  • JSON diagnostic output via --diagnostic-format.
  • New CLI flags: --strict-diagnostics, --pass-stats, --fast-link, --time-compile, --paranoid-verify, --debug-lines/--no-debug-lines.
  • VM step-over/step-out debugger and typo suggestions on undefined identifiers.

Compiler throughput

  • Parallel IL optimizer and codegen — Hardware-concurrency thread pool drives both optimization and code generation passes.
  • Zero-copy codegen pipeline — Verified IL module passed directly to the backend, eliminating the serialize → temp-file → re-parse round-trip.
  • Analytical AArch64 instruction sizing and bitset-backed liveness for reduced allocator overhead.
  • Fast-link mode auto-enabled at -O0; zanna run defaults to debug profile for quick script invocations.

Packaging & distribution

  • Six-pass hardening of zanna install-package and zanna package: Windows long-path support, ProgID tagging, PATH deduplication, PE security flags (ASLR/NX/CFG/HEVA), manifest embedding validation, PNG full color-type support, ZIP central-directory cross-check.
  • macOS signed app packaging with ad-hoc, Developer ID, hardened runtime, notarization, and stapling support.
  • Full /// @brief Doxygen pass across all 20+ packaging source files.

Platform fixes

  • macOS — Premultiplied RGBA unpremultiply, stuck-key fix on Cmd+Tab, surrogate-pair text input, long-uptime mach_absolute_time overflow fix.
  • Linux — Full ICCCM CLIPBOARD protocol, XDND percent-decoding, XLookupString overflow retry, ConfigureNotify resize bounds validation, extended cursor shape mapping.
  • All platforms — vgfx_cls() strict-aliasing UB fix, Cohen-Sutherland pre-clip, Bresenham int64 accumulator, atomic event queue.

By the numbers

  • Production SLOC: ~450K → ~552K (+102K)
  • Source files: 2,869 → 2,996 (+127)
  • Test SLOC: ~183K → ~228K (+45K)
  • Commits: 140 since v0.2.4

Demos shipped

Text-mode baseball franchise simulator; Crackman rewrite; two new 3D demos; Paint gains layers and undo; ZannaIDE file-watcher and context-menu fixes; XENOSCAPE boss/player fixes and macOS package; Chess drag-vs-click detection fix; three localization examples; Windows ARM64 builds for all 3D demos.

# Zia Language Audit — Bugs & Doc Mismatches (2026-03-29)

This file tracks all issues found during systematic verification of every
documented Zia language feature against the actual compiler implementation.

Status key: **BUG** = compiler/runtime defect, **DOC** = documentation wrong,
**MISSING** = documented but not implemented

---

## P0 — Crashes / Data Corruption

### BUG-ZIA-001: String bracket-index crashes heap allocator — **FIXED**
- **Repro**: `var ch = "Hello"[0]; Say(ch);`
- **Error**: `Assertion failed: (hdr->magic == RT_MAGIC), function payload_to_hdr, file rt_heap.c`
- **Cause**: Lowerer had no String-specific case in `lowerIndex()`, fell through to List path.
- **Fix**: Added String case in `Lowerer_Expr_Collections.cpp` that emits `Substring(base, index, 1)`.
- **Tests**: 1361/1361 pass after fix.

## P1 — Wrong Behavior

### BUG-ZIA-002: List[Boolean] indexing returns i64 instead of i1 — **FIXED**
- **Repro**: `var bools = [true, false]; SayBool(bools[0]);`
- **Error**: `call arg type mismatch: @Viper.Terminal.SayBool parameter 0 expects i1 but got i64`
- **Cause**: `emitUnbox()` in Lowerer_Emit.cpp returned I64 type for boolean unboxing instead of truncating to I1.
- **Fix**: Added `Trunc1` instruction after `kUnboxI1` call to truncate i64→i1.
- **Tests**: 1361/1361 pass after fix.

### BUG-ZIA-003: `catch(e)` binding is always empty string — **FIXED**
- **Repro**: `try { throw "hello"; } catch(e) { Say("got: " + e); }` → prints `got: `
- **Root causes**: (1) `lowerThrowStmt` discarded the thrown value with `(void)result;` (2) No mechanism to pass the message to catch handlers (3) Catch variable was bound to opaque Error pointer, not a string.
- **Fix**: Three-part fix:
  1. Added `rt_throw_msg_set`/`rt_throw_msg_get` C runtime functions (thread-local string storage in rt_error.c)
  2. `lowerThrowStmt` now coerces thrown value to string and calls `Viper.Error.SetThrowMsg`
  3. Catch variable binding calls `Viper.Error.GetThrowMsg` and stores as String type
  4. Sema changed catch variable type from `Ptr` to `String`
- **Tests**: 1361/1361 pass after fix.

### BUG-ZIA-004: `String.Contains()` returns void — **FIXED**
- **Repro**: `var r = "hello".Contains("ell");`
- **Error**: `instruction type must be non-void`
- **Cause**: String class had `Has` method but no `Contains` alias in runtime.def.
- **Fix**: Added `RT_METHOD("Contains", "i1(str)", StrHas)` to String class in runtime.def.
- **Tests**: 1361/1361 pass after fix.

### BUG-ZIA-005: `Math.Abs()` returns f64 but docs imply integer version
- **Repro**: `SayInt(Abs(-42));`
- **Error**: `call arg type mismatch: expects i64 but got f64`
- **Cause**: `Abs()` in `Viper.Math` returns f64. Need `AbsInt()` for integers.
- **Affected docs**: Ch 13 (Math section shows `Abs()` without distinguishing int/float)

## P2 — Missing Features (Documented but Not Implemented)

### MISSING-ZIA-001: `func(...)` lambda syntax
- **Documented**: `var add = func(a: Integer, b: Integer) -> Integer { return a + b; };`
- **Error**: `expected expression` — `func` not recognized as expression
- **Status**: Only arrow syntax `(params) => expr` works.
- **Affected docs**: Appendix A, zia-reference.md

### MISSING-ZIA-002: Variadic parameters (`...Type`)
- **Documented**: `func sum(numbers: ...Integer) -> Integer { ... }`
- **Error**: `expected type` at `...`
- **Status**: Not implemented in parser.
- **Affected docs**: Appendix A

### MISSING-ZIA-003: Single-expression function syntax (`= expr;`)
- **Documented**: `func double(x: Integer) -> Integer = x * 2;`
- **Error**: `expected function body`
- **Status**: Not implemented in parser.
- **Affected docs**: Appendix A

### MISSING-ZIA-004: Array repeat syntax (`[value; count]`)
- **Documented**: `var zeros = [0; 10];`
- **Error**: `expected ], got ;`
- **Status**: Not implemented. Use loop with Push instead.
- **Affected docs**: Ch 06

## P3 — Documentation Gaps (Features Missing from Docs)

### DOC-GAP-001: `SayInt()`, `SayNum()`, `SayBool()` not documented in Bible
- These typed print functions are essential for printing non-string values.
- Bible Ch 02–03 only document `Say()` and `Print()`.
- `Say(integerVar)` works via auto-concat, but direct `Say(42)` needs `SayInt`.

### DOC-GAP-002: `catch(e)` vs `catch e` vs `catch` syntax confusion
- `catch(e) { }` — parses, runs, but `e` is always empty (BUG-ZIA-003)
- `catch e { }` — parse error
- `catch { }` — works, no variable binding
- Docs should document only `catch { }` until BUG-ZIA-003 is fixed.

### DOC-GAP-003: `IndexOf()` is 1-based, not 0-based
- Bible Ch 08 implies 0-based (consistent with array indexing).
- Actual behavior: returns 1-based position, 0 means "not found".
- This is a surprising API choice that must be prominently documented.

### DOC-GAP-004: String interpolation `${}` works but not documented in zia-reference.md
- The main reference doc mentions it briefly but Bible Ch 03 doesn't cover it at all.
- It's a major convenience feature that beginners should learn early.

### DOC-GAP-005: Math functions need int/float suffixes
- `Abs`, `Min`, `Max` are f64. Integer versions: `AbsInt`, `MinInt`, `MaxInt`, `ClampInt`.
- Bible Ch 13 and Appendix A only document `Abs`/`Min`/`Max` without distinguishing.

## P2 — More Missing Features

### MISSING-ZIA-005: Type aliases (`type Name = Type`)
- **Documented**: `type UserId = String;`
- **Error**: `Unknown type: type`
- **Status**: Not implemented in parser.
- **Affected docs**: Appendix A

### MISSING-ZIA-006: Shift operators (`<<`, `>>`)
- **Documented**: `1 << 4`, `16 >> 2`
- **Error**: `expected expression` at `<`
- **Status**: Lexer/parser doesn't recognize `<<`/`>>` as operators.
- **Affected docs**: Appendix A, zia-reference.md (operator precedence table)

### MISSING-ZIA-007: Shift-assign operators (`<<=`, `>>=`)
- **Documented**: In compound assignment table.
- **Status**: Not implemented (follows from MISSING-ZIA-006).

### MISSING-ZIA-008: Bitwise compound assignment (`&=`, `|=`, `^=`)
- **Documented**: In compound assignment table.
- **Status**: Untested but likely not implemented.

### MISSING-ZIA-009: Static members need `expose`
- **Tested**: `static func increment()` — parses but inaccessible externally.
- **Error**: `Cannot access private member 'increment' of type 'Counter'`
- **Fix**: Likely needs `expose static func` or `static expose func`.
- **Affected docs**: Appendix A, Ch 14

### MISSING-ZIA-010: Namespace `final` constants
- **Tested**: `namespace Utils { final VERSION = 42; }` then `Utils.VERSION`
- **Error**: `Undefined identifier: Utils` (for constant access)
- **Note**: `Utils.double(5)` function call works, but constant access doesn't.

### MISSING-ZIA-011: `is` operator returns false for parent types
- **Tested**: `var d = new Dog(); d is Animal;` returns `false`
- **Expected**: `true` (Dog extends Animal, so Dog IS-A Animal)
- **Status**: `is` only checks exact type, not inheritance hierarchy.

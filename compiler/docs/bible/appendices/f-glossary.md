# Appendix F: Glossary

Key terms and concepts used throughout this book.

---

## A

**Abstract class**: A class that cannot be instantiated directly; serves as a base for other classes.

**Algorithm**: A step-by-step procedure for solving a problem or performing a computation.

**API (Application Programming Interface)**: A set of functions and protocols for building software; defines how components interact.

**Argument**: A value passed to a function when calling it. See also *parameter*.

**Array**: An ordered collection of elements, accessed by numeric index (starting at 0).

**Assignment**: Storing a value in a variable using the `=` operator.

**Asynchronous**: Operations that don't block execution while waiting; the program can continue doing other work.

**Atomic operation**: An operation that completes entirely or not at all, with no intermediate states visible to other threads.

---

## B

**Binary**: Base-2 number system using only 0 and 1; also refers to compiled executable files.

**Block**: A group of statements enclosed in braces `{ }`.

**Boolean**: A type with only two values: `true` or `false`.

**Bug**: An error in a program that causes incorrect behavior.

**Byte**: 8 bits; the basic unit of data storage.

---

## C

**Cache**: Temporary storage for frequently accessed data to improve performance.

**Call stack**: The stack of function calls that led to the current point of execution.

**Channel**: A communication mechanism for sending data between threads.

**Class**: A blueprint for creating objects, defining their data and behavior.

**Closure**: A function that captures variables from its enclosing scope.

**Compiler**: A program that translates source code into executable code.

**Concatenation**: Joining strings together end-to-end.

**Concurrency**: Multiple tasks making progress over time, possibly interleaved.

**Conditional**: A statement that executes code based on whether a condition is true.

**Constant**: A named value that cannot change after it's set.

**Constructor**: A special method called when creating a new object.

**CPU (Central Processing Unit)**: The processor that executes program instructions.

---

## D

**Data structure**: A way of organizing and storing data (arrays, maps, trees, etc.).

**Data type**: A classification that specifies what kind of values a variable can hold.

**Deadlock**: A situation where threads wait for each other indefinitely, unable to proceed.

**Debugging**: Finding and fixing errors in a program.

**Declaration**: Introducing a new variable, function, or type to the program.

**Dependency**: Code that your code relies on to function.

**Dependency injection**: Passing dependencies to a component rather than having it create them.

**Dynamic dispatch**: Selecting which method to call at runtime based on the actual object type.

**Dynamic typing**: Type checking done at runtime rather than compile time.

---

## E

**Encapsulation**: Hiding internal details of a class, exposing only what's necessary.

**Enumeration (enum)**: A type consisting of a fixed set of named values.

**Exception**: An error condition that disrupts normal program flow.

**Expression**: Code that evaluates to a value (e.g., `2 + 3`, `x * y`).

---

## F

**Float/Floating-point**: Numbers with decimal points (e.g., 3.14, -0.5).

**Framework**: A reusable structure that provides a foundation for building applications.

**Function**: A reusable block of code that performs a specific task.

---

## G

**Garbage collection**: Automatic memory management that reclaims unused memory.

**Generic**: Code that works with multiple types, specified as parameters.

**Git**: A version control system for tracking changes in code.

**Global variable**: A variable accessible from anywhere in the program.

---

## H

**Hash**: A fixed-size value computed from data, used for quick lookups and verification.

**Hash map/table**: A data structure that maps keys to values using hashing for fast lookup.

**Heap**: Memory region for dynamically allocated objects.

**HTTP (Hypertext Transfer Protocol)**: The protocol for web communication.

---

## I

**IDE (Integrated Development Environment)**: A software tool for writing and debugging code.

**IL (Intermediate Language)**: Viper's internal representation between source code and execution.

**Immutable**: Cannot be changed after creation.

**Inheritance**: A class adopting properties and methods from a parent class.

**Instance**: A specific object created from a class.

**Integer**: A whole number without a decimal point.

**Interface**: A contract specifying methods a class must implement.

**Interpreter**: A program that executes code directly, line by line.

**Iteration**: Repeating a process; one pass through a loop.

---

## J

**JSON (JavaScript Object Notation)**: A text format for structured data.

---

## K

**Keyword**: A reserved word with special meaning in the language (e.g., `if`, `func`, `class`).

---

## L

**Lambda**: An anonymous (unnamed) function, often used inline.

**Library**: A collection of reusable code.

**Literal**: A fixed value written directly in code (e.g., `42`, `"hello"`, `true`).

**Local variable**: A variable only accessible within a specific scope (function, block).

**Loop**: Code that repeats until a condition is met.

---

## M

**Map**: A collection of key-value pairs; also called dictionary or hash map.

**Memory**: Storage where programs keep data while running.

**Method**: A function that belongs to a class or object.

**Module**: A self-contained unit of code that can be imported.

**Mutex (Mutual Exclusion)**: A synchronization primitive ensuring only one thread accesses a resource.

**Mutable**: Can be changed after creation.

---

## N

**Namespace**: A container that groups related code and prevents naming conflicts.

**Native code**: Machine code that runs directly on the CPU.

**Null/Nil**: A special value representing "no value" or "nothing."

---

## O

**Object**: An instance of a class, containing data and methods.

**Object-oriented programming (OOP)**: A paradigm organizing code around objects.

**Operator**: A symbol that performs an operation (e.g., `+`, `-`, `==`).

**Override**: Replacing a parent class method with a new implementation in a child class.

---

## P

**Package**: A collection of related modules.

**Parallelism**: Multiple tasks executing simultaneously on different CPU cores.

**Parameter**: A variable in a function definition that receives an argument value.

**Parse**: To analyze text and convert it to a structured format.

**Pattern matching**: Checking data against patterns and extracting parts.

**Pointer**: A value that stores a memory address.

**Polymorphism**: The ability to treat different types uniformly through a common interface.

**Primitive type**: A basic built-in type (integer, float, boolean, etc.).

**Private**: Accessible only within the class that defines it.

**Property**: A class member accessed like a field but backed by methods.

**Protocol**: A set of rules for communication or interaction.

**Public**: Accessible from anywhere.

---

## Q

**Query**: A request for data, often from a database.

**Queue**: A data structure where elements are added at one end and removed from the other (FIFO).

---

## R

**Race condition**: A bug where behavior depends on timing of concurrent operations.

**RAM (Random Access Memory)**: Fast, temporary computer memory.

**Recursion**: A function calling itself.

**Refactoring**: Restructuring code without changing its behavior.

**Reference**: A value that points to an object rather than containing it directly.

**Regex (Regular Expression)**: A pattern for matching text.

**Repository pattern**: Abstracting data access behind a common interface.

**Return value**: The value a function sends back when it completes.

**Runtime**: The period when a program is executing; also the supporting software.

---

## S

**Scope**: The region of code where a variable is accessible.

**Serialization**: Converting data to a format for storage or transmission.

**Set**: A collection of unique values.

**Singleton**: A class that allows only one instance.

**Socket**: An endpoint for network communication.

**Stack**: Memory for function calls and local variables; also a LIFO data structure.

**Stack trace**: A list of function calls leading to an error.

**Statement**: A complete instruction (declaration, assignment, control flow, etc.).

**Static**: Belonging to the class itself rather than instances.

**Static typing**: Type checking done at compile time.

**String**: A sequence of characters (text).

**Struct/Structure**: A composite type grouping related values together.

**Subclass**: A class that inherits from another class.

**Superclass**: The parent class that another class inherits from.

**Synchronization**: Coordinating concurrent access to shared resources.

**Syntax**: The rules for valid code structure in a language.

---

## T

**TCP (Transmission Control Protocol)**: A reliable network protocol for ordered data delivery.

**Test**: Code that verifies other code works correctly.

**Thread**: An independent sequence of execution within a program.

**Thread pool**: A collection of reusable threads for executing tasks.

**Thread-safe**: Code that works correctly when called from multiple threads.

**Timeout**: A limit on how long to wait for an operation.

**Token**: A basic unit of source code (keyword, identifier, literal, operator).

**Type**: A classification defining what values a variable can hold and what operations are valid.

**Type inference**: The compiler automatically determining types from context.

---

## U

**UDP (User Datagram Protocol)**: A fast but unreliable network protocol.

**Unicode**: A standard for representing text characters from all writing systems.

**Unit test**: A test for a small, isolated piece of code.

**UTF-8**: A common Unicode encoding.

---

## V

**Value type**: A type where variables hold the actual data (not a reference).

**Variable**: A named storage location for data.

**Virtual method**: A method that can be overridden in subclasses.

**Virtual machine (VM)**: Software that executes code in an isolated environment.

**Void**: Indicating a function returns no value.

---

## W

**WebSocket**: A protocol for full-duplex web communication.

---

## X

**XML (Extensible Markup Language)**: A text format for structured data using tags.

---

## Numbers

**32-bit/64-bit**: The size of values a type or system can handle (32 or 64 binary digits).

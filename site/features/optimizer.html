Optimizer Passes

Viper's IL optimizer is a full optimization pipeline with 28 transformation passes and 7+ analysis passes, all built from scratch. The pass list reads like an LLVM textbook — SCCP, GVN, LICM, Mem2Reg, loop unrolling, dead store elimination — but every line is Viper's own implementation.

Optimization Levels

O0 — Minimal (2 passes)

Basic cleanup only. Used for fastest compilation during development.

simplify-cfg → dce

O1 — Moderate (15 passes)

Default for VM-interpreted and debug builds. Runs key optimizations without aggressive transformations.

simplify-cfg → mem2reg → simplify-cfg → sccp → constfold → dce
→ simplify-cfg → inline → simplify-cfg → sccp → dce → licm
→ simplify-cfg → peephole → dce

O2 — Aggressive (29 passes)

Full optimization for production native code. Includes loop optimizations, GVN, and dead store elimination.

loop-simplify → loop-rotate → indvars → loop-unroll → simplify-cfg
→ mem2reg → simplify-cfg → sccp → check-opt → eh-opt → dce
→ simplify-cfg → sibling-recursion → inline → simplify-cfg → sccp
→ constfold → dce → simplify-cfg → licm → simplify-cfg → gvn
→ reassociate → earlycse → dse → peephole → dce → late-cleanup

Pass Reference

Core Simplification

PassDescription
SimplifyCFGEliminates empty blocks, folds trivial branches, merges blocks, prunes unreachable code
Mem2RegPromotes alloca/store/load patterns to SSA registers; includes SROA support
DCEDead code elimination via reverse-use traversal
ConstFoldEvaluates pure operations at compile time (arithmetic, bitwise, conversions)
Peephole57 algebraic simplification patterns (bitwise identities, arithmetic rules)
LateCleanupEnd-of-pipeline polish: up to 4 iterations of SimplifyCFG + DCE

Advanced Analysis-Driven

PassDescription
SCCPSparse Conditional Constant Propagation — propagates constants through the CFG, folds branches
GVNGlobal Value Numbering — cross-block redundant computation elimination
EarlyCSECommon Subexpression Elimination scoped by dominator tree
DSEDead Store Elimination with MemorySSA for precise def-use chains
CheckOptRemoves redundant safety checks (bounds, div-by-zero) using dominance analysis
InlineCost-model inlining considering instruction count, call site constants, and code growth
ReassociateExpression reassociation to enable further CSE and strength reduction

Loop Optimization

PassDescription
LoopSimplifyCanonicalizes loop structure (preheaders, dedicated exits)
LoopRotateRotates loops to improve code layout and enable further optimization
LoopUnrollFully unrolls small constant-bound loops (default: max 8 iterations, 50 instructions)
LICMLoop-Invariant Code Motion — hoists invariant operations out of loop bodies
IndVarSimplifyInduction variable optimization and strength reduction

PassDescription

Analysis Framework

The optimizer is backed by a reusable analysis framework with caching and invalidation:

AnalysisDescription
CFGControl flow graph construction
DominatorsDominator tree computation
Post-DominatorsIterative post-dominator tree on reversed CFG
LoopInfoLoop detection and nesting analysis
LivenessVariable liveness analysis
BasicAAAlias analysis with pointer classification
MemorySSAMemory SSA for precise store-to-load def-use chains
CallGraphCall graph and SCC analysis for inlining decisions

The AnalysisManager caches computed analyses and invalidates them based on PreservedAnalyses metadata returned by each transformation pass. Debug builds verify IL correctness between passes.

Backend Peephole Optimization

After IL optimization, both native backends apply additional target-specific peephole passes on the Machine IR:

  • AArch64: 7 passes including loop-invariant constant hoisting, strength reduction (div → shift), and copy propagation
  • x86-64: 5 passes including arithmetic simplification, MOV folding, and fixed-point DCE

See Native Targets for backend peephole details.

Battle-Tested

Viper's test suite has 1,984 tests across 876 test files, covering every layer of the platform. All 1,279 CTest targets pass on every commit. The project runs sanitizer CI (ASan, UBSan, TSan) and includes 5 fuzz harnesses for input surface robustness.

Fuzz Testing

Five libfuzzer-based harnesses test input surface robustness:

  • fuzz_zia_lexer — Feeds random bytes to the Zia lexer
  • fuzz_zia_parser — Feeds random token streams to the Zia parser
  • fuzz_basic_lexer — Feeds random bytes to the BASIC lexer
  • fuzz_basic_parser — Feeds random token streams to the BASIC parser
  • fuzz_il_parser — Feeds random bytes to the IL parser

Enabled via VIPER_ENABLE_FUZZ=ON. These harnesses have found and helped fix edge cases in comment handling, string literal parsing, and error recovery.

Golden Tests (~203 tests)

Golden tests capture textual stability — they compare actual output against checked-in expected output:

  • IL serialization — Ensures IL text format doesn't silently change
  • Diagnostic messages — Both compilers' error/warning messages are regression-tested
  • Optimizer output — Verifies that optimization passes produce expected IL
  • Exception handling — Try/catch lowering correctness

Golden files are updated via ./scripts/update_goldens.sh when intentional changes are made.

End-to-End Tests

E2E tests verify the complete compilation pipeline from source code through execution:

  • Frontend → IL — Both compilers produce correct IL from source
  • IL → VM — VM produces expected output for test programs
  • IL → Native — Native binaries produce expected output
  • VM vs Native — Differential testing ensures identical output (see Determinism)
  • Demo execution — Real demo programs compile and run without crashing

Sanitizer CI

Three LLVM sanitizers run on every CI build:

  • AddressSanitizer (ASan) — Detects buffer overflows, use-after-free, double-free
  • UndefinedBehaviorSanitizer (UBSan) — Detects signed overflow, null dereference, alignment violations
  • ThreadSanitizer (TSan) — Detects data races in concurrent code

Enabled via IL_SANITIZE_ADDRESS=ON, IL_SANITIZE_UNDEFINED=ON, IL_SANITIZE_THREAD=ON.
